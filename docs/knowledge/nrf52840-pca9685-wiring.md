# 知识点：nRF52840 与 PCA9685 I2C 连线

PCA9685 是 16 通道、12 位 PWM 驱动芯片，通过 **I2C** 与 MCU 通信，常用于舵机、LED 调光等场景。本文说明 nRF52840（以 **nRF52840 DK** 为例）与 PCA9685 模块的接线方式。

---

## 1. 基本连线（四线 I2C）

```
nRF52840              PCA9685
--------------------------------
3.3V          ----->  VCC
GND           ----->  GND
SDA (GPIO)    ----->  SDA
SCL (GPIO)    ----->  SCL
```

- **VCC**：PCA9685 的**逻辑电源**，接 MCU 侧 **3.3 V**（与 nRF52840 IO 电平一致）。
- **GND**：两端共地，必须连接。
- **SDA / SCL**：I2C 数据线、时钟线，**一一对应**，不可交叉。

> PCA9685 模块上通常还有 **V+**（负载电源，如舵机用的 5–6 V）和 **OE**（输出使能）。它们**不属于**与 nRF52840 的 I2C 通信线；V+ 接舵机电源正极，OE 悬空或接 GND（常使能）即可，见下文第 4 节。

---

## 2. nRF52840 DK 默认 I2C 引脚

DK 板载设备树（`nrf52840dk/nrf52840`）中，两路 I2C 默认引脚如下：

| I2C 总线 | Zephyr 节点 | SDA | SCL | 常见用途 |
|----------|-------------|-----|-----|----------|
| **I2C0** | `arduino_i2c` / `&i2c0` | **P0.26** | **P0.27** | Arduino 排针 **SDA / SCL** |
| I2C1 | `&i2c1` | P0.30 | P0.31 | 需自定义 overlay 时使用 |

**推荐**：接 DK **Arduino 兼容排针**上的 I2C 时，使用 **I2C0**：

```
nRF52840 DK           PCA9685
--------------------------------
3V3 (排针)    ----->  VCC
GND (排针)    ----->  GND
P0.26 / SDA   ----->  SDA
P0.27 / SCL   ----->  SCL
```

若使用其他 GPIO 作为 I2C，需在 Zephyr 中通过 **设备树 overlay** 重映射 `twim` 的 `psels`，不能随意换线而不改软件配置。

---

## 3. 接线示意

```text
                    nRF52840 DK
                 ┌─────────────────┐
                 │  3V3 ──────────────── VCC ──┐
                 │  GND ──────────────── GND ──┤  PCA9685
                 │  P0.26 (SDA) ──────── SDA ──┤  模块
                 │  P0.27 (SCL) ──────── SCL ──┘
                 └─────────────────┘
```

## 4. PCA9685 模块其他引脚（可选）

| 引脚 | 说明 | 建议接法 |
|------|------|----------|
| **V+** | PWM 负载电源（如舵机 **5–6 V**） | 外接舵机电源正极，**不要**从 nRF52840 的 3.3 V 取电 |
| **GND** | 逻辑地与负载地 | 与 DK、舵机电源**共地** |
| **OE** | 输出使能，低电平有效 | 不用时接 **GND**（常开输出）；或接 GPIO 做开关 |
| **A0–A5** | I2C 地址配置 | 默认全接 GND → 7 位地址 **0x40** |
| **PWM0–15** | 16 路 PWM 输出 | 接舵机信号线或 LED 等负载 |

逻辑侧（VCC / SDA / SCL）与负载侧（V+ / PWM）电源应**分开供电、共地**，避免大电流舵机从 3.3 V 拉电导致 MCU 复位。

---

## 5. I2C 电气与软件要点

### 5.1 上拉电阻

I2C 为开漏总线，**SDA、SCL 各需上拉**到 3.3 V（常用 **2.2 kΩ–4.7 kΩ**）。

- 多数 PCA9685 成品模块（如 Adafruit 16-Channel PWM）**板载已带上拉**，可直接与 DK 相连。
- 若用裸芯片或无上拉模块，需在 SDA、SCL 上各加一只上拉到 3.3 V。

### 5.2 设备地址

- 默认 7 位地址：**0x40**（A0–A5 全低）。
- 通过 A0–A5 接 VCC/GND 可配置 **0x40–0x7F** 范围内地址；多片 PCA9685 同总线时需错开地址。

### 5.3 Zephyr 配置示例（`prj.conf`）

```ini
CONFIG_I2C=y
CONFIG_GPIO=y
```

应用层通过 `DEVICE_DT_GET(DT_NODELABEL(i2c0))` 获取 I2C0 设备（与 P0.26/P0.27 对应）。

### 5.4 通信速率

PCA9685 支持标准模式（100 kHz）与快速模式（400 kHz）。初次联调建议 **100 kHz**，确认通信正常后再提高速率。

---

## 6. 上电检查清单

1. **3.3 V → VCC**，**GND 共地**，无反接。
2. **SDA ↔ SDA**、**SCL ↔ SCL**，未与 UART/SPI 引脚混用。
3. 舵机等大电流负载接 **V+**，不从 DK 3.3 V 供电。
4. 用万用表/逻辑分析仪确认 SDA、SCL 空闲时为高电平（上拉有效）。
5. I2C 扫描或读 PCA9685 模式寄存器（地址 **0x40**）验证通信。

---

## 7. 与 ble-projects 工程的关系

- 板型默认：`nrf52840dk/nrf52840`（见 `docs/setup.md`）。
- 引脚定义来源：`sdk/ncs/zephyr/boards/nordic/nrf52840dk/nrf52840dk_nrf52840-pinctrl.dtsi`。
- 若改接 **I2C1（P0.30 / P0.31）** 或其他 GPIO，在对应项目的 `boards/nrf52840dk_nrf52840.overlay` 中覆盖 `&i2c0` / `&i2c1` 的 `pinctrl-0`。
