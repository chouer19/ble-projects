# Kame Motion Framework 使用说明

> 8DOF Mini-Kame 四足机器人统一动作框架。
>
> 架构：**动作序列（Motion Sequence）+ 状态机（FSM）**。
> 不追求实时步态、不使用 IK、不做轨迹规划。
>
> 相关源码与文档：
> - `include/kame_pose.h` / `src/kame_pose.c` — Pose（静态姿态）抽象
> - `include/kame_motion.h` / `src/kame_motion.c` — Motion 枚举 + 步序表 + FSM 线程
> - `src/kame_servo_cal.c` — 舵机标定与方向（见 `kame-servo-calibration.md`）
> - [architecture.md](architecture.md) — 软件架构与线程模型
> - [progress.md](progress.md) — 开发进度与已知限制

---

## 1. 概念与分层

```text
shell:  <动作> [腿]   (顶层命令，无前缀)
          │
          ▼
kame_motion (FSM)  ── 步序表 motions[]，后台线程每拍推进一个 Step
          │  产出
          ▼
kame_pose          ── 一个 Pose = 8 舵机目标角(按 servo_id)；STAND/TRIPOD/LIFT…
          │  kame_servo_apply(servo_id, deg)   ← 抽象访问，禁止写死通道号
          ▼
kame_servo_cal     ── servo_id → 通道/标定/方向；hip_forward / knee_up 定向偏移
          ▼
main.c 后端        ── 50Hz 插值线程 → PCA9685
```

- **Pose**：整机一帧姿态，8 个舵机的目标角度（按 `servo_id` 索引）。
- **Step**：一个 Motion 的一个动作帧，执行后输出一个 Pose 并等待固定时间。
- **Motion**：由多个 Step 组成的动作。同一时刻**仅运行一个** Motion，允许随时切换。
- **节拍**：每个 Step 之间等待 `motion_step_delay_ms`（默认 **500ms**）。

### 腿定义

```text
        前 (Front)
   L1 ──────────── R1
   │                │
   │     机身       │
   │                │
   L2 ──────────── R2
        后 (Rear)
```

| 腿 | 位置 | 髋关节 servo | 腿关节 servo |
|----|------|--------------|--------------|
| L1 | 前左 | L1/hip | L1/knee |
| R1 | 前右 | R1/hip | R1/knee |
| L2 | 后左 | L2/hip | L2/knee |
| R2 | 后右 | R2/hip | R2/knee |

每条腿 2 个关节：`hip`（髋）+ `knee`（腿/膝）。

## 2. 串口命令总览

> 命令已精简：**动作直接作为顶层命令，无 `spider motion` 前缀**，便于串口快速输入。
> （`spider scan/check/servo/angle/...` 等舵机调试命令仍保留在 `spider` 下。）

```text
<动作> [腿]
```

| 命令 | 类型 | 参数 | 说明 |
|------|------|------|------|
| `speed [ms]` | — | 毫秒 | 查看/设置节拍 `motion_step_delay_ms`（默认 500） |
| `stop` | — | — | 停止当前动作并**恢复立正(STAND)** |
| `stand` | 静态 | — | 标准站姿 |
| `splay` | 静态 | — | 叉腿站立（前腿前 / 后腿后） |
| `lift <腿>` | 静态 | L1/R1/L2/R2 | 抬起指定腿并**保持**（先进 TRIPOD） |
| `front_down` | 静态 | — | 前身趴下 |
| `rear_down` | 静态 | — | 后身趴下 |
| `full_down` | 静态 | — | 整体趴下 |
| `forward` | 循环 | — | 前进 |
| `backward` | 循环 | — | 后退 |
| `turn_left` | 循环 | — | 弧线左转 |
| `turn_right` | 循环 | — | 弧线右转 |
| `spin_left` | 循环 | — | 原地左旋 |
| `spin_right` | 循环 | — | 原地右旋 |
| `nod` | 循环 | — | 点头 |
| `shake` | 循环 | — | 摇头 |
| `roll` | 循环 | — | 横滚摆动 |
| `wave <腿>` | 循环 | L1/R1/L2/R2 | 招手，一直摆动（先进 TRIPOD；`stop` 恢复立正） |

> - **静态动作**：跑完全部 Step 后停在末态（如 `lift` 停在抬腿姿态）。
> - **循环动作**：跑完后回到 Step0 反复执行，需 `stop` 或切换其他动作来停止。
> - **stop 行为**：`stop` = 立即恢复立正(STAND)，对任意动作均适用。
> - **切换规则**：发任意新动作命令会立即打断当前动作。任意手动舵机命令
>   （`spider angle/servo/multi/all/move/stand/splay/off/offall`）也会自动停止当前动作，避免争用。

## 3. 节拍配置（motion_step_delay_ms）

```text
speed          # 查看当前节拍
speed 500      # 默认 500ms
speed 300      # 加快（每个 Step 间隔 300ms）
speed 800      # 放慢
```

- 节拍只控制 **Step 之间的等待时间**；舵机本身在 Step 内的移动速度由全局速度
  `spider speed <deg/s>` 控制。
- 若节拍过短而舵机还没到位就进入下一步，会出现“追不上”的现象 —— 适当增大节拍或
  提高 `spider speed`。

## 4. 单腿动作与三脚架（TRIPOD）

**所有单腿动作（`lift`、`wave`）执行前必须先进入稳定三脚架姿态，禁止直接抬腿。**

`TRIPOD_L1 / TRIPOD_R1 / TRIPOD_L2 / TRIPOD_R2` 不是独立命令，而是由
`kame_pose_tripod(leg)` 参数化生成，并被 `lift` / `wave` 复用。它让**其余三条腿**
形成稳定支撑：

- **主：调整三条支撑腿的髋关节**平移机身重心（四脚着地时旋转髋会让机身相对脚掌前后平移）：
  - 抬**前腿**(L1/R1) → 三条支撑腿髋向「前」转 → 机身后移 → 重心落入后侧支撑三角；
  - 抬**后腿**(L2/R2) → 三条支撑腿髋向「后」转 → 机身前移 → 重心落入前侧支撑三角。
- **辅：对侧压腿 + 同侧收腿**，横向把重心推入支撑三角。
- 目标腿本身保持站立角，待下一步再抬起。

### LIFT 流程（静态，2 步，停在末态）

| Step | 动作 |
|------|------|
| 0 | 进入对应 TRIPOD 姿态（支撑腿调整髋/膝形成稳定支撑） |
| 1 | 抬起目标腿，**保持** |

> 抬腿是静态动作：发出后目标腿抬起，其余三腿保持稳定支撑，**停在该姿态**。
> 结束请用 `stop`（恢复立正）或切换其他动作。

示例：

```text
lift L1
lift R2
```

### WAVE 流程（循环）

| Step | 动作 |
|------|------|
| 0 | TRIPOD 稳定，目标腿膝关节**贴到最高端** |
| 1 | TRIPOD 稳定，目标腿膝关节**自最高端回落**（最高端 − 摆幅） |
| … | 循环 0/1，目标腿在**最高点附近**膝关节上下摆动，其余三腿保持稳定支撑 |

> 招手是循环动作：目标腿膝关节在**各自最高点附近**往复摆动（像挥手，停在高处），
> 髋关节保持站立角不动。摆动区间为 `[最高端 − KAME_WAVE_KNEE_DEG, 最高端]`
> （如 L1 膝 0~180°、180° 最高 → 在 130~180° 之间摆）。
> 只要不停止就一直招手；发 `stop` 直接恢复立正(STAND)。

示例：

```text
wave L1      # 开始招手（循环）
stop         # 停止并恢复立正
```

## 5. 循环步态原理

离散对角步态（无 IK、无轨迹规划）：

- **对角配对**：A = {L1, R2}，B = {R1, L2}
- **4 拍循环**：

| 拍 | 抬起 | 髋姿态 |
|----|------|--------|
| 0 | A | A 摆到「前」，B 在「后」 |
| 1 | — | A 落地于前 |
| 2 | B | B 摆到「前」，A 在「后」 |
| 3 | — | B 落地于前 |

落地腿由「前」拖向「后」即为驱动行程，推动机身前进。

> **外张偏置（防撞）**：前腿（L1/R1）恒偏向机身前、后腿（L2/R2）恒偏向机身后
> （`GAIT_HIP_BIAS_DEG`），前后摆动叠加在该偏置上。因 `BIAS > GAIT_HIP_DEG`，
> 前腿永不越过中位向后、后腿永不越过中位向前，**同侧前后腿摆动窗口分处中线两侧、
> 永不相向靠拢**，从根本上避免同侧前后腿（L1↔L2、R1↔R2）互撞损坏舵机。

每条腿一个方向系数 `dir`（+1 向前 / −1 向后 / 0 原地）即可派生不同运动：

| 动作 | L1 | R1 | L2 | R2 | 效果 |
|------|----|----|----|----|------|
| forward | +1 | +1 | +1 | +1 | 全腿向前 |
| backward | −1 | −1 | −1 | −1 | 全腿向后 |
| turn_left | 0 | +1 | 0 | +1 | 左侧原地、右侧前进 → 弧线左转 |
| turn_right | +1 | 0 | +1 | 0 | 右侧原地、左侧前进 → 弧线右转 |
| spin_left | −1 | +1 | −1 | +1 | 左侧后/右侧前 → 原地左旋 |
| spin_right | +1 | −1 | +1 | −1 | 原地右旋 |

机身姿态摆动类（2 拍循环）：

- **nod**：两前腿膝关节上下往复（点头）。
- **shake**：两前腿髋关节反向往复（机身前部偏摆，摇头）。
- **roll**：左右两侧膝关节交替抬/压（机身横滚）。

## 6. 静态姿态说明

| 姿态 | 说明 |
|------|------|
| STAND | 按标定表的标准站姿 |
| SPLAY | 叉腿站立：髋关节前腿向前、后腿向后偏移（膝关节保持站立角） |
| FRONT_DOWN | 前两腿膝关节收起 → 前身落下 |
| REAR_DOWN | 后两腿膝关节收起 → 后身落下 |
| FULL_DOWN | 四腿膝关节收起 → 整体趴下 |

## 7. 可调参数

> 框架的步幅 / 抬腿量 / 倾斜量等均为命名常量，便于在硬件上微调。修改后重新
> `make build PROJECT=spider` 即可。

### `include/kame_pose.h`

| 宏 | 默认 | 含义 |
|----|------|------|
| `KAME_SPLAY_DELTA_DEG` | 30 | 叉腿髋偏移 |
| `KAME_TRIPOD_HIP_DEG` | 25 | 三脚架：支撑腿**髋关节**调整，平移机身重心离开抬起腿（主）；同时是抬腿侧前后腿的靠拢量，受防撞约束 |
| `KAME_TRIPOD_LEAN_DEG` | 25 | 三脚架：对侧压腿 + 同侧收腿，横向把重心推入支撑三角（辅） |
| `KAME_LIFT_KNEE_DEG` | 75 | 抬腿：目标腿膝关节向最高方向偏移 |
| `KAME_WAVE_KNEE_DEG` | 50 | 招手：目标腿膝关节自最高端向下回落的摆幅 |
| `KAME_DOWN_KNEE_DEG` | 45 | 趴下：相关腿膝关节收起幅度 |

### `src/kame_motion.c`

| 宏 | 默认 | 含义 |
|----|------|------|
| `GAIT_HIP_DEG` | 20 | 步态髋前后摆幅（单侧；以外张偏置为中心前后各摆该角） |
| `GAIT_HIP_BIAS_DEG` | 25 | 外张偏置：前腿恒偏前、后腿恒偏后，防同侧前后腿互撞（需 > `GAIT_HIP_DEG`） |
| `GAIT_LIFT_DEG` | 40 | 摆动腿抬起量 |
| `ROLL_DEG` | 25 | 横滚幅度 |
| `NOD_DEG` | 30 | 点头（前腿膝）幅度 |
| `SHAKE_DEG` | 20 | 摇头（前腿髋）幅度；前腿后摆会靠近同侧后腿，受防撞约束 |
### 节拍

| 参数 | 默认 | 修改方式 |
|------|------|----------|
| `motion_step_delay_ms` | 500 | `speed <ms>` |

## 8. 典型操作流程

```text
# 上电后会自动执行 stand 复位到标准站姿（无需手动输入）。
# 如需重新站立，仍可手动发送：
stand

# 调慢节拍便于观察
speed 700

# 抬腿并保持（自动先进三脚架），结束用 stop 恢复立正
lift L1
stop

# 招手（循环，一直招手），stop 恢复立正
wave R1
stop

# 行走（循环，stop 退出）
forward
stop

# 原地旋转
spin_left
stop

# 趴下休息
full_down
```

## 9. 注意事项

1. 所有动作代码统一通过 `servo_id` / `leg_id` 访问，**不出现 PCA9685 通道号**；
   通道号只存在于标定表 `kame_servo_cal.c` 与底层 `main.c`。
2. 单腿动作严格遵循「先三脚架后抬腿」，请勿绕过框架直接抬腿；三脚架以**调整支撑腿
   髋关节**平移重心为主、膝关节侧倾为辅。
3. `lift` 为静态动作，抬腿后保持；`wave` 为循环动作，一直招手。两者结束都用 `stop`
   恢复立正。
4. `spin/turn` 旋向、`tripod` 重心平移量为基于几何的默认值，首次上机请用较慢节拍
   （如 `speed 800`）观察并微调第 7 节参数（重点 `KAME_TRIPOD_HIP_DEG`）。
6. 循环动作不会自动停止，务必用 `stop` 或切换动作结束。
