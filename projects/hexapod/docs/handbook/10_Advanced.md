# 第十章 高级动作

高级动作是六足从「会走会演」走向「会应对地形与模仿生物」的能力层。它们风险更高、参数更密、对感知与估计更敏感，因此默认带 ADV_ 前缀思维：可演示、可降级、可急停，不默认对儿童开放。本章覆盖爬坡下坡、越障、跨台阶、自动找落脚点、自动水平、自动恢复，以及蜘蛛、狗、昆虫、螃蟹、猫等步态气质模拟，并扩展螳螂威慑、寄居蟹缩壳、壁虎攀附感、马急行感、变色龙慢移、蚂蚁搬运协作感、蟑螂受惊逃窜克制版、节肢动物脱皮舒展。

设计总则：先稳后炫；斜坡与台阶必须有倾角与边缘策略；模仿生物是「气质编码」不是解剖学复制；所有高级动作在失稳时让位于自动恢复；禁止在本章文档中依赖底层寄存器直控。实现须遵守 [第十三章 AI 规范](13_AI_Guidelines.md) 与 [第十一章 动作库](11_Motion_Library.md)。

## 爬坡 Ascent

**名称**：爬坡（Slope Ascent）

**MotionID**：ADV_SLOPE_ASCENT

**星级**：★★★★★

**场景**：坡道演示、户外缓坡、无障碍坡、训练场倾斜板。倾角超过安全阈则拒绝执行并提示。

**目的**：在上坡时保持足够牵引力与抗后仰，稳定向上推进，体现六足地形优势。

**描述**：爬坡不是简单把前进指令加在斜面上。机身应略增前倾以把重心投影移向坡上方足端，步长缩短，摆动腿抬高以防磕坡面，支撑相延长以提高静稳定裕度。腿1与腿6在坡上负责探路与拉拽感，腿2与腿5提供侧向抗滑，腿3与腿4防止后坐。J2/J3 协同改变等效腿长以维持机身相对坡面的目标姿态，而不是强行保持世界系绝对水平（绝对水平由自动水平动作负责）。检测到打滑则减小步长并增大触地时间；连续打滑则停止并呼叫恢复或后退下坡策略。儿童模式仅允许极缓坡。禁止在爬坡中插入庆祝跳跃。与下坡参数对称但符号相反的部分必须单独整定，不能简单复用。

**视觉效果**：一步一稳往上拱，机身贴合坡势，少见打滑空转，看起来吃力但自信。

**流程**：评估倾角 → 切换爬坡姿态 → 短步上爬 → 周期性姿态校正 → 坡顶减速站稳。

**节奏**：慢于平地。步频降低，支撑占比升高。

**参数**：slope_angle；step_length；body_pitch_bias；slip_threshold；max_slope；gait。

**默认参数**：body_pitch_bias=随坡增；step_length=短；gait=三角稳；max_slope=产品标定。

**速度**：明确慢。坡越大越慢。

**身体姿态**：相对坡面微前倾；横滚近零；高度略降增稳。

**腿部动作**：抬腿更高；落足更狠更慢；后腿防滑蹬。

**步态**：稳三角；极陡可改爬行波。

**可组合**：前接探路；后接自动水平；失败接恢复。

**结束姿态**：坡顶或中止点对称站稳。

**进入退出条件**：进入：倾角合法且地面可通行。退出：完成、超阈、打滑、急停。

**变种**：地毯坡；湿坡禁用；展示缓坡。

**扩展**：视觉坡估计；足端力打滑观测；多坡段切换。

**APP语音自动**：APP 高级页确认。语音：爬坡。自动：导航坡段。

**第一印象**：它真的会爬山。

**值得实现理由**：六足核心卖点；地形演示必拍。

**难度**：高。

**实现顺序**：先缓坡短步，再打滑检测，再更陡降级策略。

**注意事项**：防后翻；设最大坡；禁跳；护栏；热与电流监控。 高级动作默认需要成人监护或展厅护栏；失稳立即交给恢复策略。机体参数保持一致：蜘蛛原型六足，十

**设计深化**：爬坡不是「用力往前走」，而是把身体通道变成坡度伺服。俯仰应随坡缓增，前腿探、中腿撑、后腿推形成清晰三段职责；观众应读出「谨慎上攻」而非「打滑挣扎」。四足机器人的坡面姿态规划与山地徒步的三点支撑原则都适用：任何时刻支撑多边形投影必须包含重力方向修正后的等效重心。MG996R 在持续爬升时温升更快，因此默认速度必须低于平地巡航，并允许中途 Rest 散热。失败时应后撤回平直段再 Alert，而不是在坡上原地空转。与 `MOVE_HILL_ASSIST`、`ADV_LEVEL` 的关系是：爬坡是场景技能，辅助步与水平保持是其内部子策略。


## 下坡 Descent

**名称**：下坡（Slope Descent）

**MotionID**：ADV_SLOPE_DESCENT

**星级**：★★★★★

**场景**：从坡道下来、舞台斜板下行、户外回程。比上坡更容易失速前冲。

**目的**：控制下行速度与前冲，保持可刹可停，避免翻滚与跪地。

**描述**：下坡关键是抗前冲：机身相对坡面略后仰或降低重心，步长更短，摆动腿先探再移重心，支撑腿膝关节（J3）勿突然折叠。速度钳制强于平地，必要时插入三足支撑停顿。前腿像刹车锚点，后腿控制放行。打滑时立刻减小步频并增大触地，必要时改为侧向上下的斜向横移减势。禁止下坡奔跑感马急行。与自动找落脚点结合时，每一步先验证前方落点高度。儿童仅极缓坡。结束在平地前要有姿态回正段，避免仍以后仰姿走平地。

**视觉效果**：小心翼翼下山，步伐碎，身体稳，不像轮式溜坡。

**流程**：评估倾角 → 下坡姿 → 短步探落下行 → 停顿校核 → 坡底回正站稳。

**节奏**：比爬坡更慢更碎。

**参数**：slope_angle；step_length；body_pitch_bias；brake_gain；max_speed。

**默认参数**：step_length=很短；brake_gain=高；max_speed=低。

**速度**：极慢。忽略加速。

**身体姿态**：重心偏低；相对坡微后仰；横滚锁定。

**腿部动作**：前腿锚；后腿放；抬腿防刮坡。

**步态**：稳三角或爬行波。

**可组合**：接自动水平；失败接恢复；与跨台阶区分。

**结束姿态**：平地对称站立。

**进入退出条件**：进入：合法下坡。退出：坡底、超速风险、急停。

**变种**：侧身下行；极缓展示。

**扩展**：速度闭环；视觉坡底检测。

**APP语音自动**：需确认。语音：下坡。自动：导航下行段。

**第一印象**：下坡也很稳，不吓人。

**值得实现理由**：安全闭环；与爬坡成对能力。

**难度**：高。

**实现顺序**：先极缓，再制动策略，再打滑。

**注意事项**：前冲是第一危险；禁跑；儿童严限。 高级动作默认需要成人监护或展厅护栏；失稳立即交给恢复策略。

**设计深化**：下坡的恐惧点是加速滑坠。身体应略后仰、步幅缩短、落足更「放」而不是「砸」。动画上的刹车步与汽车下坡点刹同构：用多次短支撑消耗势能。儿童与展厅默认禁止大倾角下坡表演；若必须演示，护栏与人员监护是产品条件而非可选项。与后退、警戒后退组合时，优先保持面朝下坡方向的可控定向，避免横向失稳。结束必须在平地 Stand 稳定窗口达标后才允许接下一个花式动作。

**场景补充（下坡）**：下坡验收看三件事：会不会越走越快、会不会横滑、停得下不停得稳。默认把速度上限设为平地的六到七成，并把 step_height 略降以减少「踩空感」。若 IMU 显示俯仰超过阈值，应自动插入更短的点刹步或直接转自动恢复。展厅演示应用斜坡道具标注角度，避免观众误以为平地也能这么「飘」。与喜剧性的沉重步相反，下坡美学是克制与专业。

## 越障 Obstacle Traverse

**名称**：越障（Obstacle Traverse）

**MotionID**：ADV_OBSTACLE_TRAVERSE

**星级**：★★★★☆

**场景**：低门槛、线缆垄、短砖条、训练场障碍条。高障碍应拒绝或改绕行。

**目的**：用抬腿与机身起伏越过低障碍，保持其余足支撑，体现灵活。

**描述**：越障策略：接近减速→前足探测高度→判断可越/应绕→可越则提高摆动腿轨迹并在越障瞬间微升机身→逐足越过→回落。一次只解放最少腿数，优先三角支撑。障碍高于阈值则停止并提示绕行，不盲目抬。线缆类应避免勾挂，落足避开回弹物。与跨台阶不同，越障强调单条障碍两侧等高。模仿昆虫高抬腿气质可在此复用，但安全优先于形似。禁止用跳跃越障作为默认。

**视觉效果**：一条腿一条腿认真跨过障碍，像谨慎的大昆虫。

**流程**：Approach → Probe → Decide → Lift-traverse per leg group → Settle。

**节奏**：慢，探测停顿清楚。

**参数**：obstacle_height；clearance；probe_enable；abort_to_bypass。

**默认参数**：probe_enable=开；abort_to_bypass=开。

**速度**：慢。

**身体姿态**：越障瞬间微升；前后保持俯仰可控。

**腿部动作**：高抬；落点远障碍；支撑锁死。

**步态**：断续步态，非连续巡航。

**可组合**：接寻找绕行；失败恢复；可与自动找落脚点联动。

**结束姿态**：障后对称站立。

**进入退出条件**：进入：障碍已知或探测。退出：完成/绕行/急停。

**变种**：展示高抬；家庭地毯边。

**扩展**：深度相机高度图；勾挂检测。

**APP语音自动**：高级确认。语音：越过障碍。

**第一印象**：它会自己跨过去。

**值得实现理由**：家庭实用；演示灵活。

**难度**：高。

**实现顺序**：先已知高度越障，再探测，再绕行决策。

**注意事项**：防勾；防超高强越；护线。 高级动作默认需要成人监护或展厅护栏；失稳立即交给恢复策略。机体参数保持一致：蜘蛛原型六足，十八自由度，每腿 J1 基节横摆、J2 大腿俯仰、J3 膝；舵机 Tower Pro MG996R×18；Home 为 J1=90°、J2=180°、J3=125°（悬空标定，不等于

**设计深化**：越障是「探针—决策—跨越—确认」的句子，不是跳高。先 `LEG_PROBE`/`LEG_TOUCH_GROUND` 建立障碍可读性，再抬高机身与摆动腿，跨越后天数步软着陆。游戏中的 vault 与昆虫攀越都强调预备可见；本机无踝，只能靠膝折与胯摆制造净空。自动越障若感知不确定，必须停并 Observe，而不是赌一把。产品包上应默认关闭，创作者/比赛包再解锁。

**场景补充（越障）**：越障的产品叙事是「它会看路」。即使第一版感知很粗，只要探针与停顿做满，观众就愿意相信后续会更聪明。越障成功后给一个很小的 Level Up 或点头，能把工程能力翻译成情感奖励。反之，失败时的 Confused + 后退比强行跨越更能建立信任。注意障碍边缘的足端刮擦会损坏贴纸地板，展厅地面材质要进验收清单。

## 跨台阶 Step Climb

**名称**：跨台阶（Step Climb / Stair Step）

**MotionID**：ADV_STEP_CLIMB

**星级**：★★★★☆

**场景**：单级台阶、低台阶训练、门口石阶。多级楼梯需逐级确认，默认不连续自动爬楼。

**目的**：把足端依次送到更高平面并转移重心，完成单级上台或下台。

**描述**：跨台阶是离散高度变化：上台时前足踏上，中足跟进，后足推升，机身随阶抬升；下台反向锚定。每步都要验证目标平面存在，避免踩空。自动找落脚点在此成为内建依赖。台阶边缘是最大风险，足端应落在踏面安全区内而非棱上。多级楼梯必须一级一停，允许取消。儿童与展厅默认禁用真台阶，只用极低演示块。与爬坡不同，台阶有突变。失败立即冻结并恢复。

**视觉效果**：一级级认真爬，停顿清楚，不像莽撞跳台阶。

**流程**：Measure step → Place front feet → Shift → Mid feet → Rear bring-up → Stabilize。

**节奏**：极慢，步间停顿。

**参数**：step_height；tread_depth；safe_margin；direction；max_steps。

**默认参数**：max_steps=1；safe_margin=充足；direction=上或下。

**速度**：极慢。

**身体姿态**：随阶改变高度；俯仰短时变化后回目标。

**腿部动作**：高抬上台；探深下台；始终多足支撑。

**步态**：非周期专用序列。

**可组合**：强依赖自动找落脚点；失败自动恢复。

**结束姿态**：目标平面站稳。

**进入退出条件**：进入：台阶参数合法。退出：完成、踩空风险、急停。

**变种**：仅演示块；下台阶版。

**扩展**：连续楼梯人工确认模式；视觉棱线检测。

**APP语音自动**：强确认。语音：上台阶。自动：默认关。

**第一印象**：六足爬阶很震撼。

**值得实现理由**：能力上限展示；工程难点突破口。

**难度**：很高。

**实现顺序**：先低块，再真单阶，再谨慎多阶。

**注意事项**：踩空；棱边；儿童禁用；必须可取消。机体参数保持一致：蜘蛛原型六足，十八自由度，每腿 J1

**设计深化**：台阶是离散高程，比连续坡更吃「单腿承重时间」。每级应有：探边→支撑重组→上踏→收腿→稳定。对称左右交替能减少观众晕眩感。禁止两级并作一次；失败回退比强行更像专业设备。与自动找落脚点联动时，台阶前沿是最高优先级特征。

**场景补充（跨台阶）**：台阶高度若接近小腿长度的显著比例，必须降为「一级一确认」。儿童场景禁用自动跨台阶。教学演示可把每一级拆成慢动作，配合讲解员口播支撑腿编号，变成六足运动课。对称的 1/6、2/5、3/4 轮换能保持机头朝向稳定，这是「像动物上楼梯」而不是「像叉车颠簸」的关键。

**长期维护笔记（跨台阶）**：本技能入库后应同时具备手册条目、MotionID、默认参数表与产品包标记（child_safe / exhibit_safe / creator_only）。实机验收固定三机位（侧视看俯仰与高度、俯视看支撑与对称、斜视看整体气质），并用「三米外一秒可读」做门禁。所有轨迹经 Motion 层调度，支持 speed、中断、暂停、恢复；禁止 PWM/PCA9685 直控。中断点默认放在稳定支撑相，中断后优先 Stand 或 Safe Hold。与相邻技能的边界要写进评论：谁调用谁、谁互斥、谁在失稳时让位给自动恢复。舵机温升、地毯/硬地差异、左右镜像一致性（1↔6、2↔5、3↔4）列入回归清单。若只在单一地面调美，不得标记为 ★★★★★。家庭默认取保守幅度，表演档单独解锁并提示净空与监护。文档、APP 文案与语音词条共用同一中文名，避免「会做但用户找不到」。

## 自动找落脚点

**名称**：自动找落脚点（Foothold Planning）

**MotionID**：ADV_FOOTHOLD_PLAN

**星级**：★★★★★

**场景**：不规则地面、台阶、碎石演示（受控）、越障与爬坡的内建能力层。

**目的**：为摆动腿选择稳定、可及、无碰撞的落点，而不是盲目落回名义轨迹。

**描述**：自动找落脚点是高级地形的大脑之一。根据局部高度图或探触结果，在候选触地区域内评分：平坦度、支撑力估计、与当前支撑多边形组成的稳定裕度、运动学可及性、防自碰。选中后生成足端轨迹。无感知时退化为「探触—试探—确认」行为：轻触、加载、再移重心。它通常不单独表演，而被跨台阶、越障、爬坡调用；但展厅可做可视化慢动作演示。选点失败则缩小步长或停止。必须有超时与安全默认。六足候选点还要考虑左右对称与中足特殊位置。

**视觉效果**：落脚前有探或短停，落得很「有考虑」，减少滑擦。

**流程**：Sense/candidates → Score → Select → Swing trajectory → Load test → Commit。

## 自动恢复 Auto Recover

**名称**：自动恢复（Auto Recover）

**MotionID**：ADV_AUTO_RECOVER

**星级**：★★★★★

**场景**：失稳、摔倒倾向、腿悬空、急停后混乱姿态、演示推扰后自救。

**目的**：尽快回到可站稳对称姿态，保护舵机与结构，再交给上层。

**描述**：自动恢复是安全网。检测异常倾角、足端卸载异常或控制栈求助后：停止原动作→保护性降功率冲击→按脚本收腿/伸腿尝试建立三点以上支撑→撑起→自动水平→待命。若多次失败，进入最小安全姿并呼叫人工。禁止恢复过程做表演性舞蹈。推扰演示必须在海绵护栏环境。恢复脚本要考虑侧倾与仰倾不同。儿童模式更早呼叫人工、更少激进撑起。与组合急停衔接：急停先冻结，再决定是否恢复。

**视觉效果**：慌一下然后自己站住，观众鼓掌；失败则老实趴稳等救。

**流程**：Detect → Freeze → Support seek → Push-up → Level → Idle/Help。

**节奏**：越快越好但有上限，避免猛撑断齿。

**参数**：max_attempts；help_timeout；gentle_mode；post_level。

**默认参数**：max_attempts=3；post_level=开；gentle_mode=儿童开。

**速度**：保护性中慢，禁止猛抽。

**身体姿态**：从乱到稳到水平。

**腿部动作**：先找支撑再同步撑；避免单腿暴载。

**步态**：专用恢复序列。

**可组合**：全局可抢占；成功后回行为层。

**结束姿态**：对称站立或安全低姿+求助。

**进入退出条件**：进入：失稳事件。退出：站稳、求助、急停保持。

**扩展**：跌倒方向分类脚本；日志黑匣子。

**APP语音自动**：自动；APP显示恢复中。语音：站起来（人工触发温和版）。

**第一印象**：摔不坏，会自救。

**值得实现理由**：安全基石；敢做高级动作的前提。

**难度**：很高。

**实现顺序**：先安全低姿，再撑起，再水平，再多姿态分类。

**注意事项**：防二次损伤；限流；护栏；勿无限重试。

**设计深化**：恢复是安全网与信任组件。优先级高于一切表演：检测到支撑不足、异常倾角或碰撞后，停止花式，降高度，重组三角/多足支撑，回到 Stand 或 Safe Hold。工业故障安全与游戏 hitstun 起身都要求「先能站，再谈帅」。恢复日志必须带原因码，供 APP 显示「我稳住了」。

**视觉效果**：吓一跳跑两步又站住，喜剧效果好且可控。

**流程**：Startle → Turn away → Short dash → Hard brake → Assess → Embarrassed recover。

**节奏**：逃短，刹硬，总长二到五秒加恢复。

**参数**：dash_steps；speed_cap；child_noflee；cooldown。

**默认参数**：dash_steps=3..5；child_noflee=开；cooldown=长。

**速度**：短时中高后强制低。

**身体姿态**：逃时压低；刹后升。

**腿部动作**：快步短。

**步态**：短促三角。

**可组合**：戏剧；其后缩壳或道歉若撞到。

**结束姿态**：观察站立或壳。

**进入退出条件**：显式使能。

**变种**：儿童无逃；纯喜剧夸张刹停。

**扩展**：声源方位逃避。

**APP语音自动**：双重确认。语音慎用。

**第一印象**：哈哈哈太逼真——还好停得住。

**值得实现理由**：应激研究科普；喜剧内容；刹车工程。

**难度**：高（安全）。

**实现顺序**：先无位移受惊，再一步逃，再多步+刹。

**注意事项**：距离帽；儿童；勿对人群；冷却；急停。

**设计深化**：短促 Zigzag/Dash 后必须 Freeze/Soothe，且默认关闭。只作创作者/生物课对比「惊逃 vs 礼貌后退」。

**场景补充（蟑螂受惊逃窜克制版）**：此动作默认隐藏。启用时必须：短、可预期方向、结束后立刻安抚或定格解说「这是克制版生物逃逸」。它的教学价值是对比 Retreat Guard 的礼貌后退，而不是追求吓尖叫。

**长期维护笔记（蟑螂受惊逃窜克制版）**：本技能入库后应同时具备手册条目、MotionID、默认参数表与产品包标记（child_safe / exhibit_safe / creator_only）。实机验收固定三机位（侧视看俯仰与高度、俯视看支撑与对称、斜视看整体气质），并用「三米外一秒可读」做门禁。所有轨迹经 Motion 层调度，支持 speed、中断、暂停、恢复；禁止 PWM/PCA9685 直控。中断点默认放在稳定支撑相，中断后优先 Stand 或 Safe Hold。与相邻技能的边界要写进评论：谁调用谁、谁互斥、谁在失稳时让位给自动恢复。舵机温升、地毯/硬地差异、左右镜像一致性（1↔6、2↔5、3↔4）列入回归清单。若只在单一地面调美，不得标记为 ★★★★★。家庭默认取保守幅度，表演档单独解锁并提示净空与监护。文档、APP 文案与语音词条共用同一中文名，避免「会做但用户找不到」。

## 节肢动物脱皮舒展

**名称**：节肢动物脱皮舒展（Arthropod Molt Stretch）

**MotionID**：ADV_MIMIC_MOLT

**星级**：★★★★☆

**场景**：唤醒后深层拉伸、养护仪式、语音「舒展一下」、生物蜕皮科普。

**目的**：用缓慢逐肢伸展与躯干起伏模拟蜕皮后的舒展，传达新生与养护感。

**描述**：脱皮舒展是慢表演：机身微升→按腿序 1-6 或对称对依次缓伸到近行程柔边再收回→全身轻抖收尾→回站。每腿伸展时其他腿锁定。可与唤醒例程深层版衔接，但本动作更长更「养护」。禁止到硬限位砸死。展厅可配解说蜕皮。结束后像更轻快一点的站立参数短暂生效（可选）。

**视觉效果**：一条腿一条腿醒来伸直，很有节肢动物纪录片感。

**流程**：Rise → Sequenced leg extend/retract → Soft whole-body shake → Fresh stand。

**节奏**：很慢，总长二十到四十五秒。

**参数**：leg_order；extend_pct；hold；shake；post_fresh_boost。

**默认参数**：extend_pct=柔边内；hold=0.5s；shake=小。

**速度**：慢。

**身体姿态**：中高；伸展时微补偿。

**腿部动作**：单腿大行程缓动；支撑锁。

**步态**：无。

**可组合**：唤醒后；养护日程；脱皮科普节目。

**结束姿态**：清新站立。

**进入退出条件**：指令/养护。

**变种**：短版三条腿；超长冥想版。

**扩展**：与舵机养护行程活动共用。

**APP语音自动**：语音：舒展一下、蜕皮。

**第一印象**：像刚蜕完皮，好治愈。

**值得实现理由**：养护+科普；全行程柔控练兵。

**难度**：中。

**实现顺序**：先单腿，再序列，再轻抖。

**注意事项**：柔限位；防支撑不足；时间长注意场合。

**设计深化**：慢伸展、逐腿轮替、轻颤结束，像养护仪式。适合开箱后首次校准秀与「新生」剧情，切忌暴力拉满行程。

**场景补充（节肢动物脱皮舒展）**：脱皮舒展是慢美学与全行程柔控考试：逐腿伸到软限附近再释放，像在换新甲。开箱校准秀、养护日、生日「新生」剧本都适合。过程中禁止急停以外的花式插入，以免卡在单腿高举。

**长期维护笔记（节肢动物脱皮舒展）**：本技能入库后应同时具备手册条目、MotionID、默认参数表与产品包标记（child_safe / exhibit_safe / creator_only）。实机验收固定三机位（侧视看俯仰与高度、俯视看支撑与对称、斜视看整体气质），并用「三米外一秒可读」做门禁。所有轨迹经 Motion 层调度，支持 speed、中断、暂停、恢复；禁止 PWM/PCA9685 直控。中断点默认放在稳定支撑相，中断后优先 Stand 或 Safe Hold。与相邻技能的边界要写进评论：谁调用谁、谁互斥、谁在失稳时让位给自动恢复。舵机温升、地毯/硬地差异、左右镜像一致性（1↔6、2↔5、3↔4）列入回归清单。若只在单一地面调美，不得标记为 ★★★★★。家庭默认取保守幅度，表演档单独解锁并提示净空与监护。文档、APP 文案与语音词条共用同一中文名，避免「会做但用户找不到」。
