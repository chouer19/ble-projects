# 第十二章 APP设计

本章讨论 Hexapod 控制 APP 的信息架构、控件选型、儿童模式、展厅模式、语音、收藏组合、安全确认、反馈动效与无障碍。所有建议都给出「为什么」——交互选择必须服务于：**可理解、可打断、不可绕过 Motion 安全层**。

> APP 是动作库的「遥控器与导览册」，不是舵机调试示波器。默认用户通过 MotionID 与参数说话，而不是通过脉冲宽度说话。

---

## 12.1 设计总则

1. **急停永远可达**：任何页面、任何模态框、任何表演全屏，都必须能在一指之内触发急停。  
2. **动作即语义**：按钮文案用人类可读名，底层绑定 MotionID。  
3. **场景决定可见性**：儿童 / 展厅 / 开发者看到的不是同一套动作全集。  
4. **反馈先于炫技**：状态（执行中 / 暂停 / 故障）比装饰动画重要。  
5. **一条主路径**：首页只做「最常做的 80%」，其余进二级。

## 12.2 哪些动作适合哪种控件

<a id="control-mapping"></a>

### 12.2.1 按钮（点按 / 按住）

**适合**

- `POSTURE_*`：Home、蹲、展厅站立——目标态清晰，一点即达。  
- `EMO_*`：点头、摇头、挥手——短时 oneshot。  
- `PERF_*` 的「播放节目」入口。  
- `COMBO_*` 的一键演示。  
- 部分 `MOVE_*` 的「点动」：按住前进、松开停止。

**为什么**

按钮的认知成本最低，适合结果可预期的离散事件。姿态与情绪若做成摇杆，用户不知道「推到哪里才算到位」，反而增加误操作。

**不适合**

需要连续方向合成的全向移动（应用摇杆）；需要精细选腿的调试（应先选腿再按钮）。

### 12.2.2 摇杆

**适合**

- `MOVE_FORWARD` / 后退 / 横移 / 转向的矢量合成。  
- 「机体速度向量」类控制：前向分量→前进相关 MOVE，横向→strafe，旋转→turn。

**为什么**

步态是连续过程，用户期望「推多少走多快」。摇杆天然表达二维意图，并可映射到 `speed`（杆偏量）与方向参数。比一排前进/后退/左平移按钮更少打断注意力。

**设计要点**

- 松杆 = 停止或进入减速停止动作（仍走 Motion，不直切舵机）。  
- 儿童模式缩小最大 `speed` 映射。  
- 展示模式可改为「八向按钮」降低访客误触灵敏度——因为展厅访客常随手甩杆。

### 12.2.3 自动播放

**适合**

- `PERF_*` 开场/循环节目。  
- `COMBO_*` 见面礼、节日编排。  
- 展厅「无人值守循环」：待机姿态 → 短表演 → 回待机。  
- 带 `exhibit_safe` 的精选列表。

**为什么**

展厅与橱窗需要时间轴，而不是永远依赖人手。自动播放把「策展」从现场操作员解放出来；同时强制你只挑选可循环、可安全打断的动作。

**不适合**

- `ADV_*`、未标记 `exhibit_safe` 的大振幅动作。  
- 强依赖偶然传感器触发的 `BEH_*`（除非行为本身设计为展陈安全）。  
- 任何会持续靠近观众席的激进位移（除非场地隔离并有人员看管）。

### 12.2.4 录制

**适合**

- 标记 `records_well` 的 `MOVE_` / `EMO_` / `PERF_` 片段。  
- 用户在安全场地走的「路径+动作」时间线（记录的是 MotionID 序列与参数，而不是 PWM 波形）。

**为什么**

录制 MotionID 时间线可重放、可分享、可审核；录制底层角度采样会把标定误差、电压跌落一并固化，且无法在其他机体复现。产品层面应强调「录的是动作语义」。

**不适合**

- 调试时的抖动试角。  
- 含大量随机性的行为（除非同时记录随机种子与传感器摘要）。

### 12.2.5 分享

**适合**

- `COMBO_*` 与自动播放列表。  
- 带版本号的官方节目包。  
- 用户收藏夹（导出 JSON，仅含 MotionID + 参数 + library_version）。

**为什么**

分享语义包可以在检查 `library_version` 兼容后安全导入；分享「18 路角度录像」会在不同标定机器上撞限位。分享必须可被导入方的安全策略再次过滤（例如儿童模式去掉 `ADV_`）。

## 12.3 信息架构

<a id="information-architecture"></a>

### 12.3.1 顶层结构（推荐）

```text
首页（驾驶与常用）
├── 姿态
├── 表情 / 表演
├── 节目（自动播放）
├── 收藏与组合编辑器
├── 智能行为（可选）
├── 模式：儿童 / 标准 / 展厅 / 开发者
└── 设置（网络、语音、安全、关于）
悬浮：急停
全局：连接状态、电池、当前 MotionID、播放态
```

### 12.3.2 为什么这样拆

- **首页专注驾驶**：用户打开 APP 的第一意图通常是「动起来或停下来」，不是浏览百科。  
- **姿态与表情分离**：姿态改变支撑与外形，表情是短插入；混在一页会导致误触蹲下。  
- **节目独立**：自动播放是策展对象，需要列表管理、循环开关、时段，不像表情那样点一下。  
- **模式切换放在可发现但不误触的位置**：模式改变可见动作集合，属于「安全策略」，不应和前进按钮并列。

### 12.3.3 导航深度

控制类任务不超过两层到达；开发者标定可以更深。为什么：现场演示时操作者边看机器人边看手机，深层导航等于增加碰撞风险。

## 12.4 主页布局哲学

<a id="home-layout"></a>

### 12.4.1 第一屏只放五类元素

1. **连接与安全状态条**（电量、链路、模式徽章）  
2. **急停**（巨大、高对比、固定位置）  
3. **主驾驶控件**（摇杆或方向组）  
4. **3～6 个常用动作快捷钮**（可自定义，默认：Home、待机、挥手、停止）  
5. **当前动作字幕**：`正在执行：MOVE_FORWARD · speed 0.4`

### 12.4.2 为什么克制

主页不是动作商店。动作一多，访客会「每个都点一下」，展厅极易混乱。快捷钮应来自收藏与场景默认，而不是 catalog 全量。

### 12.4.3 视觉层次

- 急停：最高优先级，不参与「好看但低对比」的品牌实验。  
- 摇杆区：大而稳，避免与表情按钮抢同一热区。  
- 装饰插画可以有，但不得覆盖控件命中区。  
- 当前 MotionID 对普通用户可显示中文名，开发者模式附加 ID。

### 12.4.4 横竖屏

优先竖屏单手；展厅平板可横屏把「节目列表」与「实况字幕」并排。为什么：家长抱孩多用单手；展厅支架多用横屏展示说明。

## 12.5 儿童模式

<a id="child-mode"></a>

### 12.5.1 可见性策略

- 仅 `child_safe` 动作。  
- 隐藏全部 `ADV_`、默认隐藏 `LEG_`/`DUAL_` 调试。  
- `MOVE_*` 允许，但 `speed` 上限强制夹取（例如 ≤ 0.4）。  
- `intensity` 上限同步夹取，避免「慢但振幅巨大」。

### 12.5.2 交互策略

- 摇杆阻尼更强、中回死区更大——为什么：儿童手部抖动大，小死区会导致机器人不停点头挪步。  
- 危险确认改为「不可到达」而非「多点两次」——为什么：儿童会把双确认当成游戏连点。  
- 语音词表只用短词：前进、停下、打招呼——为什么：长句识别差，误触发表演节目风险高。  
- 录制/分享默认关闭或只允许家长解锁——为什么：避免误把现场节目表清空或导入未知组合。

### 12.5.3 家长门

进入标准/开发者模式需要简单门禁（系统口令或长按确认）。为什么：模式本身是安全边界，门禁保护的是边界，不是内容付费。

## 12.6 展厅模式 UI

<a id="exhibit-mode"></a>

### 12.6.1 目标用户

讲解员、志愿者、路过访客（只读或少操作）。不是调参工程师。

### 12.6.2 UI 特征

- **大按钮、少数量**：开场、循环演示、待机、急停。  
- **自动播放面板置顶**：当前节目、下一节目、循环开关、静音讲解（若有）。  
- **访客锁定**：可切到 Kiosk，仅留「一下个节目」与急停；为什么：防止观众把机器人走下站台。  
- **字幕友好**：中文动作名 + 一句解说；可投屏。  
- **禁用沙盒与进阶**；站点 overlay 可再砍位移距离。

### 12.6.3 为什么展厅要单独模式

展厅光照、围栏、观众距离与家庭地面完全不同。同一套「标准模式全功能」会在噪音与拥挤下被误触。展厅模式是**产品化的风险收束**，不是换皮。

### 12.6.4 展陈时间轴建议

```text
POSTURE_EXHIBIT_STAND
  → PERF_OPENING
  → COMBO_DEMO_HELLO
  → BEH_IDLE_LOOK（短）
  → PERF_LOOP_SHORT
  → 回到 POSTURE_EXHIBIT_STAND
```

任一时刻急停 → 安全停机姿态 → 人工复位后才允许继续自动播放。

## 12.7 语音词表建议

<a id="voice-lexicon"></a>

### 12.7.1 设计原则

- 词表映射到 MotionID + 预设参数，不映射到「把 J2 调到某某度」。  
- 每个词条声明置信度阈值；低置信度只提示不执行。  
- 同音冲突要避开（例如过度短的单字）。  
- 「停下 / 停止 / 急停」必须最高优先级，且在本地可离线触发（若架构允许）。

### 12.7.2 推荐核心词表（示例）

| 语音 | MotionID / 行为 | 备注 |
|------|-----------------|------|
| 停下、停止、急停 | 安全停止 / E-Stop | 永远有效 |
| 前进、往前走 | `MOVE_FORWARD` | 儿童限速 |
| 后退 | `MOVE_BACKWARD` | |
| 左转、右转 | `MOVE_TURN_LEFT/RIGHT` | |
| 回家、复位、Home | `POSTURE_HOME` | |
| 站好、待机 | `POSTURE_EXHIBIT_STAND` 或项目待机 ID | 展厅常用 |
| 打招呼、挥手 | `EMO_WAVE_RIGHT` 或 `DUAL_FRONT_WAVE` | |
| 点头 | `EMO_NOD` | |
| 开始表演 | 当前节目 `PERF_*`/`COMBO_*` | 展厅 |
| 暂停、继续 | pause / resume | 非换动作 |

### 12.7.3 为什么要小词表

识别开放域聊天会把「好厉害」误触发动作。展厅嘈杂时，封闭词表 + 唤醒词远比「全能语音助手」可靠。

### 12.7.4 多语言

中文为主；英文别名进 `aliases`。儿童模式不用中英混杂长句。

## 12.8 动作收藏与组合编辑器

<a id="favorites-combo-editor"></a>

### 12.8.1 收藏

- 收藏的是「MotionID + 默认参数预设」，不是截图。  
- 可分组：家里、展厅、拍摄。  
- 为什么：同一 `MOVE_FORWARD` 在拍摄时要慢、在展厅循环要中速，预设比让用户每次拧滑条更快且更安全。

### 12.8.2 组合编辑器

能力：

1. 按顺序添加子动作（仅已 released 且当前模式可见）。  
2. 编辑每步 `speed` / `intensity` / `duration`。  
3. 插入「等待」逻辑步（仍由 Motion 调度层执行等待，UI 不算私自 sleep 偷跑）。  
4. 预估总时长与风险标签汇总（若含非 exhibit_safe，保存为正式展厅列表时警告）。  
5. 导出为 `COMBO_USER_<id>` JSON。

### 12.8.3 为什么编辑器要限制能力

如果允许用户粘贴裸角度关键，组合编辑器会变成「绕过审核的后门」。编辑器应复用动作库合同：只能编排已有 MotionID。

### 12.8.4 预览

预览默认低 `speed`；提供「仅模拟时间轴不输出本体」的桌面模拟开关（若有模拟器）。真机预览前显示场地确认。为什么：组合错误比单动作错误更危险，因为误差会累积。

## 12.9 安全确认

<a id="safety-confirmations"></a>

### 12.9.1 急停永远可达

**要求**

- 所有页面可见（含模态、引导蒙层、视频全屏）。  
- 物理外形：圆形或圆角矩形，高对比，不与品牌主色温柔化混淆。  
- 触发后：立即对 Motion 层发 abort；UI 进入「已急停」锁存态，需明确「复位」流程。  
- 网络延迟再差，也要有本地冗余提示「指令已发出 / 未确认」。

**为什么**

没有急停的漂亮 UI 是展览事故说明书。急停是伦理底线，不是设置项。

### 12.9.2 分级确认

| 级别 | 用于 | 交互 | 为什么 |
|------|------|------|--------|
| none | 常用 EMO/MOVE（已限速） | 直接执行 | 降低摩擦 |
| soft | 单腿、较长表演 | 滑动确认或按住 0.8s | 防误触 |
| hard | `ADV_`、清空节目、导入未知库 | 双步确认 + 摘要风险 | 强制阅读 |
| gate | 退出儿童模式 | 口令/系统鉴权 | 保护边界 |

### 12.9.3 停止与急停的差异（UI 文案必须区分）

- **停止**：结束当前动作并进入安全待机（可继续）。  
- **急停**：最高优先级切断运动意图，进入锁存，需复位。  

为什么：用户把两个都叫「停」会导致演示流程卡死或安全假象。

## 12.10 反馈与动效

<a id="feedback-motion"></a>

### 12.10.1 必须反馈的状态

- 连接中 / 已连接 / 弱网  
- 动作排队 / 播放中 / 暂停 / 完成 / 被打断 / 失败  
- 安全：限位触发、重心检查失败、电池低  
- 模式：儿童 / 展厅徽章  

### 12.10.2 动效原则

- 用动效解释状态，不用动效拖延急停。  
- 按钮按下有即时按压反馈（&lt;100ms），动作本身的物理延迟用字幕与进度表达。  
- 自动播放可用时间轴进度，而不是闪烁彩虹。  
- 为什么：用户若分不清「APP 卡了」还是「机器人在慢速做动作」，就会重复点击导致指令风暴。

### 12.10.3 指令风暴防护

UI 应对连续点击做合并或忽略（尤其 oneshot）。但急停不合并、不忽略。为什么：表情按钮连点不应排 20 个挥手；急停则要每次可信。

## 12.11 无障碍

<a id="accessibility"></a>

### 12.11.1 视觉

- 不止用颜色区分急停与普通按钮（加图标与文字）。  
- 对比度满足可读；展厅强光下避免浅灰字。  
- 动态效果可关闭（前庭敏感用户）。  

### 12.11.2 运动与操作

- 所有摇杆功能提供按钮替代。  
- 最小点击区域足够大（展厅手套/紧张操作）。  
- 支持系统字体缩放，布局不崩。  

### 12.11.3 听觉

- 关键反馈可关；重要安全事件可选用短提示音，但不依赖听觉唯一通道。  

### 12.11.4 为什么无障碍不是附加分

展厅观众与家庭用户包含儿童、老人、残障人士。无障碍能直接降低误操作率，因而也是安全特性。

---

## 12.12 与动作库字段的协作

APP 应读取第十一章约定的字段：

| 字段 | APP 行为 |
|------|----------|
| `ui.control` | 决定默认控件 |
| `ui.confirm_level` | 确认门 |
| `tags.child_safe` 等 | 模式过滤 |
| `capabilities.supports_pause` | 显示暂停键 |
| `safety.needs_space` | 展厅提示 |
| `display_name` / `aliases` | 文案与搜索 |

为什么：把策略放进库，APP 才能薄；否则每个客户端各自硬编码「哪些能给儿童」会造成分裂。

## 12.13 页面级建议摘要

### 姿态页

大卡片：Home、站立、蹲、展厅待机。点按 + soft 确认（可选）。为什么：姿态改变支撑，值得比表情更谨慎的布局间距。

### 表情页

栅格按钮，执行中禁用同按钮重入（或允许打断策略随库声明）。为什么：表情应「干净」，不要叠十层抖动。

### 节目页

列表 + 循环 + 预估时长 + 风险汇总。为什么：策展需要概览。

### 开发者页

可显示 MotionID、参数 JSON、日志；仍禁止提供「写 PCA 通道」入口。为什么：开发者页是透明，不是后门。

---

## 12.14 反模式

| 反模式 | 为什么失败 |
|--------|------------|
| 急停藏在设置三级页 | 事故时找不到 |
| 儿童模式只藏图标不限 speed | 小孩仍可全速撞桌腿 |
| 主页铺满 40 个动作 | 展厅必乱 |
| 分享角度录像 | 跨机不可复现且危险 |
| 用动画挡住急停 | 直接违规 |
| 语音开放闲聊执行动作 | 误触发 |

---

## 12.15 本章小结

好的 Hexapod APP 是：**急停可信、模式收束风险、控件匹配动作时间特性、收藏组合只编排 MotionID、反馈让人知道机器在忙什么**。它消费第十一章的动作合同，并遵守第十三章的 AI/实现红线——即使按钮背后的代码是 AI 生成的。

## 12.12 跨领域控件矩阵（产品价值导向）

不要按「工程师觉得酷」排布按钮，而按用户任务：

| 用户任务 | 推荐控件 | 推荐动作包 |
|----------|----------|------------|
| 我要它过来/走开 | 摇杆 + 前进/后退限幅 | MOVE_FORWARD/BACKWARD/TURN + STOP |
| 我要打招呼 | 大按钮 | COMBO_GREETING / LEG_WAVE / PERF_BOW |
| 我想玩 | 游戏化面板 | 躲猫猫、点头摇头、庆祝 |
| 我要拍视频 | 一键剧本 | COMBO 展厅三分钟 / Birthday |
| 我要它安静 | 模式开关 | BEH 夜间静音 + Rest |
| 我要它酷 | 创作者轮盘 | 拟态、BOSS登场、蟹步（非儿童默认） |
| 出事了 | 全局急停 | MOVE_ESTOP 永远一指可达 |

### 儿童包白名单原则

默认只开放：Stand/Sit/Rest/Sleep/Wake、慢速Forward/Turn、Wave/Low Wave/Tap、Greeting/Thank/Cute、点头同意、轻量Celebrate。  
默认关闭：Dash、Kick高功率、Stomp、Fear长时、越障、BOSS、愤怒高强度。

### 展厅包原则

强调：可循环、可中断、可合影、60秒内出现高潮、结束必回Stand对位。提供「讲解员遥控」与「无人循环」两态。

### 创作者包原则

开放 Emote 轮盘、组合编辑器、方言标签过滤；导出只能是 MotionID 时间线，不能是 PWM 表。
