# 第一章 机器人运动设计思想

本章讨论的不是「如何算出一条腿的逆运动学」，而是一个更根本的问题：一台蜘蛛型六足机器人，在观众眼前站起来、迈步、停住、抬头、侧行、鞠躬时，**究竟是什么在决定它看起来像一只活物，而不是一堆在抖动的舵机**。

本手册所针对的机体是明确且固定的：蜘蛛原型、左右对称的六足步行机构；**6 条腿 × 每腿 3 自由度 = 18 DOF**；关节依次为 **J1 Hip Yaw（绕机体 +Z）**、**J2 Hip Pitch**、**J3 Knee Pitch**；全关节采用 **Tower Pro MG996R** 舵机。静止参考姿态 **Home** 为 **J1 = 90°、J2 = 180°、J3 = 125°**。机体坐标系为右手系：**+X 前、+Y 左、+Z 上**。俯视足号为：**1 左前、2 左中、3 左后、4 右后、5 右中、6 右前**。

这些参数不是装饰性背景，而是运动设计的硬边界。你设计的每一个「好看」的动作，最终都要落到这十八个关节角上；每一个「像活的」瞬间，都要经得起支撑多边形、重心投影、舵机速度与堵转扭矩的检验。本章先建立思想框架；后续章节再分别展开基础移动、姿态动作与编排细节。

## 1.1 为什么动作比算法更重要

### 1.1.1 观众看见的从来不是求解器

在工程团队内部，人们习惯用算法成熟度衡量系统先进程度：轨迹是否平滑、逆解是否稳定、步态周期是否可调、是否支持在线重规划。这些都很重要。但当机器人进入真实场景——客厅茶几旁、展厅玻璃柜前、儿童科学馆、赛场观众席——**观众的第一判断标准几乎从不包含算法名称**。

观众看见的是：

- 它站起来时，身体是稳稳升起，还是像被弹簧弹起？
- 它走路时，机身是轻微起伏而有呼吸感，还是整体在发抖？
- 它停下时，是「收住」了，还是突然冻住、像断电？
- 它转向时，是整只动物在重新定向，还是六条腿各自忙乱？

这些判断发生在数百毫秒内，属于直觉层，而不是技术层。一台逆解写得很漂亮、却在落足瞬间不断微抖的机器人，会被判为「廉价」；一台算法朴素、却把起势—加速—保持—收势做清楚的机器人，会被判为「有生命」。

因此，本手册把优先级写清楚：**动作设计是产品的第一界面；算法是动作的实现手段**。算法服务动作，而不是动作迁就算法的便利。

### 1.1.2 算法正确，动作仍可能失败

考虑一个典型反例。三角步态在数学上完全正确：支撑三角始终包围重心投影，摆动腿抬起高度足够，相位差 180°，周期恒定。按教科书实现后，机器人「能走」。但它可能仍然难看：

1. **抬腿过猛**：J2、J3 同时快速变化，足端竖直速度尖峰很高，机身被反作用力向上拽一下，看起来像受惊。
2. **落足过硬**：摆动相结束瞬间速度未清零，足端撞击地面，整机发生短时振动；MG996R 的齿轮间隙会把振动放大成可见抖动。
3. **机身无意图**：重心高度全程锁定，没有任何「准备迈步」的轻微下沉或前倾，步态像传送带，不像动物。
4. **停步无收势**：指令一停，关节立刻保持最后角度，身体悬在半个周期中间，姿态既不是站立也不是行走。

以上四点，没有一项是「公式算错」。它们是**动作语言缺失**。算法保证了可行性；动作设计保证了可读性与品质。两者缺一不可，但在产品体验上，动作缺失的伤害往往更大，因为用户不会说「你的支撑多边形算得不对」，而会说「这机器人好假」。

### 1.1.3 本机的物理现实迫使「先设计动作」

对本六足而言，动作优先还有更具体的工程理由。

**第一，执行器不是无限带宽的理想电机。** MG996R 空载约 0.14–0.17 s/60°，有间隙、有弹性、有堵转电流上限。你在轨迹层写的「瞬时速度跳变」，在硬件层会变成电流冲击、温度上升、抖动与啸叫。动作设计必须从一开始就按舵机可执行的速度包络来构思，而不是先画理想曲线再抱怨硬件跟不上。

**第二，Home 不是站立姿态。** Home 定义为 J1=90°、J2=180°、J3=125°：大腿大致竖直向上，是静止参考位与标定位，**不是触地站立位**。从 Home 到「可用站立」本身就是一段必须设计的动作，而不是一个角度赋值。若产品上电后六腿同时猛甩到站立角，观众看到的第一印象就是暴力与危险。

**第三，18 DOF 的表达力极大，出错面也极大。** 六足可以同时做机身平移、俯仰、横滚、偏航与局部腿姿调整。若没有动作思想约束，算法工程师很容易「因为能做所以全做」，结果是噪声多于信息：腿在动、身在晃、节奏却没有主题。动作设计的职责，正是在巨大可能性中选出**少数清晰意图**。

### 1.1.4 判断标准：什么叫「动作比算法更重要」落地了

可用下列可观察标准自检（不依赖任何具体 API）：

| 检查项 | 合格表现 | 不合格表现 |
|--------|----------|------------|
| 上电第一印象 | 缓慢自检、有序站起、短暂静止 | 六腿同时甩动、撞地、啸叫 |
| 行走可读性 | 三米外能判断前进/侧行/转向 | 只能看出「在抖」 |
| 停止品质 | 停在稳定站立或明确结束姿态 | 停在半周期、机身倾斜未收 |
| 失败可见性 | 动作失败时仍像「谨慎停下」 | 失败时像失控散架 |
| 重复观感 | 同一动作十次，节奏一致 | 时快时慢、偶然抖动主导观感 |

若算法指标全部达标，但上表多项不合格，说明团队仍在用算法思维做产品，尚未进入运动设计思维。

## 1.2 为什么机器人需要「性格」

### 1.2.1 性格不是拟人噱头，而是一致性策略

「性格」一词容易被误解为给机器人编造故事：它叫什么名字、喜欢什么颜色。本手册所说的性格，含义更工程化：

> **性格 = 在全部公开动作中，长期保持一致的运动偏好与情绪色调。**

具体可体现为：

- **速度偏好**：偏慢而稳，还是偏快而利落？
- **重心高度偏好**：习惯高站（更警觉、更轻盈），还是低站（更稳、更敦实）？
- **抬腿风格**：抬得高而夸张（舞台感强），还是贴地而克制（家庭安全感强）？
- **停顿习惯**：动作之间是否留「呼吸空拍」？
- **对称使用**：是否强调左右镜像的仪式感，还是允许轻微不对称以增加有机感？
- **对打扰的反应**：被中断后是立刻僵住、平滑减速，还是先完成一个安全收势？

当这些偏好在开机、待机、行走、表演、告警中反复出现，观众就会形成稳定印象：「这台机器人沉稳」「这台机器人活泼」「这台机器人谨慎」。没有性格的机器人，不是中性，而是**碎片化**——每个功能模块各写各的动作，产品像由六个陌生人拼起来。

### 1.2.2 六足天然适合「非人形性格」

人形机器人常被要求微笑、挥手、点头，观众用人类社会礼仪去衡量它。六足不同。蜘蛛型结构本身携带强烈生物联想：多足、低重心、侧向能力、停住时的「蹲伏感」。这是优势，不是负担。

本机的性格设计应主动利用结构语言：

- **J1 的水平朝向变化**适合表达「注意方向」与「重新定向」——六足转向时，不必模仿人类转腰，可以用腿基节的重新排布形成「躯体朝向改变」。
- **J2/J3 的折叠深度**适合表达「警觉高度」——机身升高像探查，降低像防御或蓄力。
- **三角步态的交替支撑**适合表达「坚定前行」；波浪步态适合表达「谨慎探测」；螃蟹侧行适合表达「横向挪移、舞台走位」。

性格不是外贴贴纸，而是从这些结构词汇中选词，并长期复用。

### 1.2.3 性格必须可被参数化描述

为避免空谈，建议每个产品版本写一份「性格卡」（文档级，而非代码级），至少包含：

1. **一句话人设**：例如「谨慎而礼貌的展览讲解助手」或「节奏分明的赛场表演者」。
2. **速度标尺**：慢/中/快三档各自用于哪些场景；禁止在儿童近距离互动中使用「赛场快」。
3. **振幅标尺**：抬腿高度、机身俯仰角、侧倾角的推荐上限。
4. **静默礼仪**：待机时是否允许轻微「呼吸」起伏；多久做一次极小的姿态重标定。
5. **情绪映射表**：欢迎、思考、成功、失败、低电、障碍——各对应哪类动作族（点头族、后退族、蹲伏族等）。
6. **禁止清单**：例如禁止突然六腿同时离地跳跃式站起；禁止在家庭场景做大幅度横滚表演。

性格卡的价值在于：产品经理改需求、动画设计师加动作、算法工程师做实现、AI 编码模型生成轨迹时，**都有同一份审美宪法可引用**。没有性格卡，团队会不断用个人口味覆盖上一个人的口味，手册无法维护。

### 1.2.4 性格与安全的关系

性格不是可以压过安全的借口。「活泼」不等于「突然加速到舵机极限」；「沉稳」不等于「慢到像故障」。正确关系是：

- 安全边界先画死（电流、倾覆、夹手空间、急停）。
- 性格在安全边界内选择风格。
- 当场景切换（家庭→展厅→比赛），先切换性格配置，再切换动作库，而不是只改一个最大速度旋钮。

对本机尤其重要的是：**儿童场景下的性格必须包含「可预测性」**。孩子会把手伸进腿间；若机器人性格是「随时插入惊喜动作」，风险会上升。展览场景可以多一点舞台停顿；比赛场景可以多一点爆发；家庭场景则应把「不惊吓」写进性格第一条款。

## 1.3 为什么动作决定产品体验

### 1.3.1 体验发生在时间里，不在参数表里

产品说明书可以写：18 自由度、MG996R、可前进可侧行可转向。这些信息帮助采购决策，却几乎不形成体验。体验是用户在时间轴上连续感受的总和：

- 等待开机的三秒，机器人是安静自检，还是杂乱抖动？
- 第一次看到它走路，是感到「哇，像真的」，还是「哦，玩具」。
- 远程让它过来时，路径是否干净，停在面前时是否礼貌？
- 表演结束后，它是否回到一个可被拍照的漂亮站姿？

每一个触点都是动作问题。UI 可以精美，语音可以聪明，但若身体动作廉价，整体产品仍会被降级为「会动的玩具」。反之，动作克制而完整，即便功能集合不大，也容易被感知为「完成度高」。

### 1.3.2 动作是信任接口

家庭与儿童场景中，信任比炫技重要。信任来自可预测：

1. **启动可预测**：上电后动作顺序固定（例如：微动自检 → 缓慢站起 → 短暂停住 → 进入待机呼吸）。
2. **移动可预测**：前进就是前进，不会无故插入转圈。
3. **停止可预测**：停止命令后，机器人在明确的短时间内进入稳定站立，而不是继续滑完很长一段花哨收尾。
4. **异常可预测**：遇到障碍或力矩异常时，表现为减速、蹲低、后退，而不是剧烈挣扎。

信任一旦建立，用户愿意让机器人靠近家具、靠近孩子、靠近展品。信任一旦破裂——比如某次突然抬腿过高扫到桌沿——后续再多「智能」也难挽回。动作设计因此不只是美学问题，更是**产品风险控制**。

### 1.3.3 动作是品牌记忆

人们可能记不住你的控制周期是 20 ms 还是 50 ms，但会记住：

- 那台六足待机时轻轻起伏，像在呼吸；
- 转向前总会先把身体重心压低半拍；
- 表演结束会做一个对称的「收翼」式蹲伏，再抬头亮相。

这些就是品牌动作签名。签名不需要多，需要稳定复现。本手册建议每个产品版本沉淀 **3～5 个签名动作**（例如：开机礼、待机呼吸、礼貌点头、侧行亮相、结束鞠躬），并在所有公开演示中优先使用它们。签名动作比临时炫技更能形成长期记忆。

### 1.3.4 体验分层：功能层、观感层、情感层

可用三层模型评审任何动作提案：

| 层级 | 问题 | 失败代价 |
|------|------|----------|
| 功能层 | 是否到达目标位姿/位置？是否不倾覆？ | 任务失败、安全事故 |
| 观感层 | 是否可读、是否平滑、是否有节奏？ | 显得廉价、演示翻车 |
| 情感层 | 是否符合性格？是否让人想再看一次？ | 品牌无记忆、用户无好感 |

很多团队停在功能层就发布。本手册要求：**公开给用户看的动作，至少达到观感层；核心签名动作必须触及情感层。**

## 1.4 动作设计原则

原则不是口号，而是评审清单。下列原则对本六足长期有效；后续章节的具体动作都应能回溯到这里。

### 1.4.1 稳定性原则

**定义：** 在动作全程，重心的水平投影应落在支撑足构成的支撑多边形内（或按设计允许的短暂动态例外，并有明确恢复策略）；机身姿态变化不得把投影长期推到边界外。

对本机：

- 静态站立至少三足支撑，且三角应尽可能「胖」——过瘦的长条三角在侧向扰动下极易失稳。
- 三角步态中，摆动组离地前，支撑组必须已经形成有效三角；禁止「先抬后找支撑」的赌气式时序。
- 机身大幅俯仰/横滚时，应同步扩大足端支撑跨度或降低重心，而不是只扭身体、腿不动。

**判断标准：** 慢放录像时，任意时刻都能指出当前支撑足，并目测重心投影大致在多边形内部；人轻推机身侧向时，机器人表现为「抗一下再回来」，而不是立刻翻腿。

### 1.4.2 可读性原则

**定义：** 三米外、非专业观众应能在一秒内说出机器人「正在做什么」。

可读性依赖：

- **主意图单一**：同一时刻只有一个主叙事（前进、侧行、鞠躬、警戒），其余为辅助。
- **肢体分工清楚**：哪些腿在支撑，哪些腿在摆动，机身在贡献什么朝向变化。
- **幅度足够被看见**：过小的点头在展厅灯光下等于没有；过大的点头在家庭茶几旁又像砸桌子。需要按场景标定「最小可见幅度」与「最大舒适幅度」。

**反例：** 同时前进、扭身、抬两条前腿挥舞——信息过载，观众只看到乱。

### 1.4.3 节奏原则

**定义：** 动作必须有时间结构，而不是匀速填满时间轴。

即使总时长相同，有起势与收势的动作也显著优于恒速动作。节奏原则要求设计者明确回答：

- 哪里加速？哪里保持？哪里减速？
- 哪里留空拍？
- 快慢对比是否服务于意图（例如：警戒时先慢后突然停住）？

后续 1.6 节将展开节奏设计细节。

### 1.4.4 可中断原则

**定义：** 动作在任意合理时刻被更高优先级事件打断时，应能进入安全、可读的过渡，而不是必须播完才停，更不能直接冻结在危险构型。

对本机，可中断至少意味着：

1. 中断后优先保证支撑足不丢失；
2. 摆动腿尽量短路径落地或收回到安全姿态；
3. 机身角速度迅速衰减，避免带着余速倾覆；
4. 结束后落在「站立」「蹲伏」或「低姿防御」等少数安全终点之一。

可中断不是牺牲美感的借口。优秀设计会把「可打断点」安排在节拍边界：支撑切换前后、姿态保持段、收势前——这样中断看起来像「决定停下」，而不是「播片卡死」。

### 1.4.5 可组合原则

**定义：** 基础动作应能像词汇组成句子：站立 + 点头、行走 + 机身微俯、侧行 + 亮相抬头，而不是每个复合动作都从零做特殊轨迹。

可组合要求：

- **终点姿态集合有限且稳定**：例如标准站立、低姿、表演站立、Home 参考（非表演用）等；
- **过渡规则明确**：从 A 姿态到 B 姿态有默认过渡策略（见 1.8）；
- **资源不冲突**：一个动作占用「机身俯仰权」时，另一个动作不应同时抢同一自由度而不加仲裁。

没有可组合性，动作库会指数爆炸，手册无法维护，AI 编码模型也会生成大量一次性脚本。

### 1.4.6 安全原则

安全在动作设计中的具体化，远不止急停按钮：

- **几何安全**：腿间、腿与机身、腿与桌沿的扫掠空间；儿童手指可能进入的区域应减少快速闭合动作。
- **动力学安全**：避免多关节同时满速反向；避免长时间逼近 MG996R 堵转；避免在支撑不足时做大幅加速。
- **语义安全**：告警动作应收敛、降低，而不是夸张扑打；「生气」类拟态在儿童产品中通常应禁止或极度克制。
- **场景安全**：家庭地毯上的抬腿高度与光滑展台不同；比赛场地上的爆发在客厅不可直接复用。

**判断标准：** 设计评审必须能回答「最坏情况下这条动作会怎样」——而不是只演示最好情况。

### 1.4.7 辨识度原则

**定义：** 关键动作在完成功能的同时，应携带可被记忆的形式特征。

辨识度来源可以是：

- 独特的停顿位置（例如侧行结束前的半拍静止）；
- 独特的对称亮相（1–6、2–5、3–4 的镜像关系被有意强调）；
- 独特的重心轨迹（走路时机身轻微「8 字」或前后波浪，但幅度受控）；
- 独特的收势符号（结束时统一回到某表演站立，而不是随意站姿）。

辨识度与可读性的平衡：太怪则不可读，太普通则无记忆。对本六足，建议把辨识度主要放在**开始与结束的仪式**，中间移动保持清晰功能。

### 1.4.8 经济性原则（常被忽略）

**定义：** 能用较小关节行程与较短时间表达清楚的，不要用更大更久。

18 个 MG996R 同时大幅度运动，电流、发热、噪声、磨损都会上升，观感也容易脏。经济性要求设计者先问：「观众需要看见的最小变化是什么？」很多「高级感」恰恰来自克制：只动该动的腿，只倾该倾的角度，只停该停的半拍。

### 1.4.9 原则冲突时的裁决顺序

真实项目里原则会打架。建议默认裁决顺序：

1. **安全**
2. **稳定性**
3. **可中断**
4. **可读性**
5. **节奏 / 可组合**
6. **辨识度 / 性格表达**
7. **经济性**（在前序满足后尽量优化）

例如：为了辨识度做夸张横滚，若威胁稳定与安全，必须降级或拆成「低风险暗示版」。

## 1.5 运动美学

运动美学不是「看起来高级」的玄学，而是可观察的形式关系。对本六足，下列维度特别有效。

### 1.5.1 线条：腿是可阅读的笔触

每条腿在侧面与俯视上都形成折线：J1 决定笔触的水平朝向，J2/J3 决定折线的开合。观众虽然说不出关节名，但能感觉到：

- **开张**：机身周围空间被腿撑开，稳定、自信、舞台占位强；
- **收敛**：腿向机身收拢或膝折叠加深，谨慎、防御、准备钻过低障碍；
- **参差**：前腿与后腿开合不一致，可能表达探查或爬坡意图，也可能只是脏——取决于是否服务于可读意图。

设计站立与表演姿态时，应先画（或想象）六条腿的折线构图，再落到角度。Home 的大腿上举构图是标定用的「收纳参考」，通常不宜作为对外展示的主站姿；对外站姿应让足端触地、膝部形成清晰承力折线。

### 1.5.2 对称：左右是六足最强的形式武器

本机左右对称关系清晰：关于 X 轴，**1↔6、2↔5、3↔4**。对称动作天然好看，也天然传达「受控」。

使用建议：

- **仪式性动作**（开机礼、鞠躬、亮相）优先严格左右对称。
- **移动类动作**允许功能所需的相位反对称（三角步态本身就是两组交替），但机身姿态应尽量保持意图清晰的对称基线，再叠加前进所需的轻微俯仰。
- **刻意不对称**只能作为短暂点缀（例如用 1 号腿轻点地面表示「指向」），必须很快回到对称基线，否则观众会解读为故障。

### 1.5.3 重心轨迹：看不见的「舞步中心线」

机身几何中心附近的等效重心，其三维轨迹是运动美学的隐藏主角。

- **水平轨迹**：直行时应接近直线或极浅的正弦；侧行时沿 +Y/−Y；转向时为平滑弧。杂乱折线 = 观感差。
- **高度轨迹**：完全锁死会像滑轨；过大起伏会像颠簸车。推荐在行走中使用「浅呼吸式」高度波动：支撑切换附近轻微下沉蓄力，摆动中段轻微回升。
- **投影与支撑多边形的相对位置**：投影长期贴边会让人潜意识紧张；投影在多边形中心附近则安心。

评审方法：用侧视与俯视慢放，只盯机身标记点，忽略腿。若标记点轨迹已经干净，腿即使略有瑕疵也常能被原谅；若标记点乱跳，腿再努力也难救。

### 1.5.4 肢体语言：意图要先于位移

动物运动的普遍规律是：**姿态变化宣告意图，位移完成意图**。六足也应遵守。

例子：

- 准备前进：先有极短的机身前倾或重心前移，再出腿。
- 准备后退：先收敛前腿或机身微后倾，再退。
- 准备侧行：先把目标侧的腿略微打开或机身微侧倾「让路」，再进入螃蟹步。
- 准备鞠躬：先低头（俯仰），再可能伴随前腿弯曲加深，而不是六腿同时无方向地蹲。

没有预动作的位移，像物体被拖拽；有预动作的位移，像主体在决定。这是「像活的」与「像被遥控的积木」的分水岭。

### 1.5.5 舞台感：空间占位与停顿

舞台感不等于大声夸张。它来自三件事：

1. **占位**：动作开始前，机器人是否「站住了」——脚位稳定、机身朝向明确、有短暂静止。
2. **释放**：主动作是否在时间上被让出来——中间段落清楚，不被前后杂音吞没。
3. **收住**：结束后是否回到可被观看的构图，并保持足够长的静止，让观众的眼睛完成一次「拍照」。

展厅与比赛尤其需要舞台感；家庭场景则要把舞台感降为「礼貌感」：占位仍要有，但幅度更小、声音更低、停顿更短。

### 1.5.6 噪声与颤振也是美学问题

MG996R 在快速换向、带载启动、间隙往复时会产生可闻噪声与可见微颤。美学评审必须把「听感」纳入：

- 同幅度动作，能更慢更顺就不要更响；
- 避免无意义的高频小修正（看起来像帕金森式抖动）；
- 表演段落宁可略降速度，也要换取干净静止帧。

一台安静、停得住的六足，往往比一台又快又响的六足更显贵。

## 1.6 节奏设计

节奏是时间艺术。六足动作若只有空间形状没有时间形状，仍会显得机械。

### 1.6.1 四段式基本句式：起势、加速、保持、收势

几乎所有可公开的动作都可以用四段句式书写：

1. **起势（Preparation）**  
   目的：宣告意图、建立支撑、收集松弛。  
   表现：短暂停顿、重心微调、相关腿预加载、机身微倾。  
   失败相：直接开跑，像被推倒后的补救。

2. **加速（Acceleration / Attack）**  
   目的：进入主运动。  
   表现：速度从低到高，主关节贡献主要位移。  
   失败相：一开始就满速，电流尖峰与视觉惊吓同时出现。

3. **保持（Sustain）**  
   目的：让观众看清主形状，或维持周期性步态的巡航段。  
   表现：速度相对稳定，形状可读。对点头、亮相等一次性动作，「保持」可能是最高点附近的短暂停；对行走，「保持」是稳定周期巡航。  
   失败相：没有巡航段，全程在加速或减速，观众抓不住主题。

4. **收势（Release / Settlement）**  
   目的：消耗余速、回到稳定构图、给出结束信号。  
   表现：减速、落足缓冲、机身回正、短静止。  
   失败相：播完最后一帧立刻接下一个命令，像电影胶片被扯断。

**建议比例（可按场景调整）：** 一次性姿态动作常用「短起势—中加速—短保持—中收势」；行走类则把「保持」拉长为周期循环，起势与收势只包在进入/退出步态的两端。

### 1.6.2 快慢对比

没有对比就没有节奏感。可用的对比手段：

- **段落对比**：慢起势 + 利落主动作 + 慢收势。
- **肢体对比**：支撑腿相对「钝」、摆动腿相对「快而轻」。
- **空间对比**：小幅度警戒摆动后，突然一个明确的大幅度侧行走位（展厅用；家庭慎用）。
- **静止对比**：在连续动作中插入半拍到一拍的完全静止——静止会放大前后动作的力量。

对比必须服务于性格。沉稳性格的对比更克制；表演性格的对比可以更戏剧。禁止为对比而对比：无动机的突然加速会被读成故障。

### 1.6.3 呼吸感

呼吸感是把「活着」从行走扩展到待机的关键技巧。

实现思想上（仍不谈实现细节）：

- 待机时，机身高度或俯仰角做极低频率、极小幅度的周期性变化；
- 六腿同步轻微调整膝角以配合高度，避免单腿单独抽动；
- 周期应接近人类安静呼吸的量级感受（偏慢），绝不能快到像振动；
- 呼吸在接到任务时应平滑退出，而不是硬切。

呼吸感的产品价值极高：它让等待不再是「死机」，而是「在场」。但呼吸过度会干扰定位与拍照——展览合影模式应提供「屏息站立」变体。

### 1.6.4 节拍与相位：六足的时间格子

三角步态天然提供二元节拍：A 组支撑 / B 组摆动交替。设计者可以把音乐性理解投射到这格子上：

- **强拍**：支撑切换完成、机身最稳的时刻——适合插入表情性机身动作（轻微点头、侧倾致意）。
- **弱拍**：摆动中段——适合完成足端前移，不宜插入大幅度机身扭动。
- **切分**：偶尔在预期强拍上保持静止，制造俏皮或警戒感——仅限表演性格，且幅度要小。

波浪步态则提供更长的「蠕动句式」，适合谨慎、探测、展示多足协调。螃蟹侧行提供横向句式，适合舞台走位与「让路」语义。

### 1.6.5 节奏的可测量评审

不依赖复杂仪器，可用这些方法：

1. **默数法**：给动作打拍子，团队是否能数出稳定节拍？
2. **静帧法**：随机暂停录像，画面是否仍像一张可发表的照片？
3. **听感法**：闭上眼只听舵机声，是否有句读，而不是一片持续尖叫？
4. **儿童法**：请非技术人员看完后用一句话复述「它在干嘛」——复述失败则节奏或可读性失败。

## 1.7 连续动作设计

### 1.7.1 从「单个技巧」到「动作句子」

产品里真正被看见的很少是孤立技巧，而是句子：

> 待机呼吸 → 抬头注意 → 三角步前进 → 减速站定 → 礼貌点头 → 回到呼吸

连续动作设计研究的是：如何让句子通顺。通顺的标准是：

- **意图链清楚**：每一段为什么接下一段，观众能猜到；
- **能量连续**：速度与机身动量没有无故跳变；
- **姿态闭合**：句子结束回到可复用的终点姿态，而不是落在一次性的奇怪构型；
- **可剪辑**：中间某些短语可被删除或替换，而不摧毁整体语法。

### 1.7.2 主题—变奏结构

长演示（展厅循环、比赛规定套路）建议采用音乐中的主题—变奏：

1. **主题呈现**：用最清晰的版本展示核心步态或签名动作；
2. **变奏**：改变方向、节奏、机身高度或加入姿态装饰；
3. **回归主题**：再次回到清晰版本，强化记忆；
4. **终止式**：专门设计的结束礼，给出「表演结束」信号。

没有终止式的长演示，观众不知道何时鼓掌，操作员也不知道何时切下一段。终止式对本六足可以很简单：侧行归位 → 对称站立 → 短静止 → 轻微低头再抬头。

### 1.7.3 动机复用

把小片段当作动机（motif）：

- 「半拍下沉」动机：出现在起步、转向前、鞠躬前；
- 「前腿打开」动机：出现在欢迎、探查、准备抓取视线；
- 「整体收敛」动机：出现在低电、障碍、结束。

动机复用会自动生成性格一致性。即使具体轨迹参数不同，观众仍觉得「还是那台机器人」。

### 1.7.4 连续动作中的信息密度控制

常见失败是信息过密：十秒内又走又转又舞又亮相。建议：

- **家庭**：低密度，一句动作说一件事；
- **展厅**：中密度，允许短复合句，但每句后要有静止；
- **比赛**：高密度，但必须有段落切分，否则评委与观众都会疲劳；
- **儿童近距离**：极低密度，优先慢、近、可预测。

### 1.7.5 循环演示的「无缝」错觉

展厅常需无限循环。无缝不是真的没有接缝，而是接缝被设计成观众不在意：

- 循环点选在对称站立静止帧；
- 进入循环与退出循环使用同一站立；
- 避免在摆动腿悬空时循环；
- 每 N 分钟插入一次极慢的「重新站稳」以消除累积误差的观感（即便控制层已有补偿，观感上仍有益）。

## 1.8 动作过渡

过渡决定连续动作是「电影」还是「幻灯片」。

### 1.8.1 姿态插值的设计含义

从一个站姿到另一个站姿，最朴素的做法是对各关节角做插值。设计手册关心的不是插值公式，而是：

- **哪些自由度应先动？** 通常先调整支撑相关的 J2/J3 以稳住高度与跨度，再改 J1 重定向，最后才做表情性机身姿态。
- **是否允许中间姿态不可展示？** 标定用的大幅度折叠可以在遮挡或低速下完成；公开过渡应尽量让每一帧都站得住、看得过。
- **时间是否均匀？** 均匀时间插值常显呆；边缘减速（慢—快—慢）更像有控制的运动。

对本机，尤其要记住：**途经 Home（J2=180° 一类上举构型）的公开过渡通常不美观**，除非你在讲「收纳/标定」叙事。对外过渡应在触地站立族姿态之间进行。

### 1.8.2 预动作（Anticipation）

预动作是起势的具体化：在主位移相反或正交的方向上，做一次短、小、可控的准备。

例子：

- 前进预动作：重心先极轻后移再前移出腿（或机身微后仰再前倾）——幅度必须很小，否则像要摔倒。
- 抬头预动作：先极轻低头再抬头，点头更有弹性。
- 侧行预动作：目标侧足稍外展，对侧稍内收，宣告「我要横向走位」。

预动作的剂量规则：**足够被感觉，不足以被误读为独立动作**。家庭场景剂量偏小；舞台场景可略大。

### 1.8.3 收尾缓冲（Settlement）

收尾缓冲处理的是「停得住」。

组成要素：

1. **速度耗散**：主运动减速到接近零；
2. **足端安顿**：所有计划触地足确实稳定承力，无足端搓动；
3. **姿态回正或定格**：回到设计终点，而不是停在最大动态偏移处；
4. **静止保持**：给出明确的结束静帧，时长按场景从 0.3 s 到数秒不等。

没有收尾缓冲的动作，即使中间再漂亮，结尾一塌糊涂，记忆也会被结尾污染——因为人类对结尾权重很高。

### 1.8.4 过渡的三种风险

| 风险 | 现象 | 设计对策 |
|------|------|----------|
| 支撑真空 | 过渡中同时抬起过多腿 | 强制过渡过程的最小支撑集 |
| 自碰与擦地 | 膝、足尖扫到邻腿或地面 | 抬腿高度下限与关节顺序约束 |
| 语义断裂 | 前一动作情绪与后一动作硬切 | 插入中性站立或短呼吸作为标点 |

### 1.8.5 「可跳过过渡」与「必须完整过渡」

不是所有过渡都同等重要：

- **必须完整过渡**：涉及支撑重组、大幅度高度变化、从行走进入表演站立等；
- **可缩短过渡**：同族姿态微调、已被中断后的紧急回稳（此时安全优先于美观）；
- **可替换过渡**：在时间压力下，用「先蹲稳再站美」两段式替换理想单段优美过渡。

把过渡分级，手册才能在真实产品约束下被执行，而不是只存在于理想演示。

## 1.9 身体重心与支撑多边形

### 1.9.1 基本几何图像

俯视机器人，将所有触地足端相连，得到支撑多边形。等效重心在水平面的投影点，记为 G′。

- **G′ 在多边形内部**：具备静态稳定基础；
- **G′ 靠近边界**：抗扰动能力变差，动作必须更慢更谨慎；
- **G′ 在外部**：静态意义上已倾覆，只能依赖动态效应或立即撑地挽救——公开产品动作应尽量避免依赖这种危险区。

对本六足，常见支撑形态包括：

- **六足站立**：多边形最大，容错最高，适合待机、讲解、近儿童；
- **三角支撑（步态）**：多边形较小，必须管理 G′ 位置与切换时序；
- **四足或五足过渡**：出现在姿态变换与故障降级中，应显式设计，而不是偶然发生。

### 1.9.2 重心高度与「稳」的观感

降低重心通常提高稳定裕度，但过低会：

- 让腹面刮地风险上升；
- 让步态抬腿空间变小；
- 让机器人显得萎靡或故障。

升高重心提升轻盈感与视野感，但侧向加速度更危险。性格卡应给出默认站立高度带，而不是只给一个点。

### 1.9.3 动态过程中的「稳定叙事」

观众不一定懂支撑多边形，但能感觉叙事是否安全：

- 抬腿前，身体是否已经「把重量交给」剩下的脚？
- 迈步中，机身是否在多边形中心附近「坐稳」？
- 落足后，是否有一瞬间的重量转移完成感？

设计行走时，应有意识地编排重量转移，而不是只编排足端轨迹。足端轨迹服务重量转移；重量转移服务稳定与美学。

### 1.9.4 俯仰、横滚、偏航对 G′ 的影响

- **俯仰（绕 Y）**：前倾使 G′ 前移，对前足要求更高；后退与下台阶时尤其危险。
- **横滚（绕 X）**：侧倾使 G′ 侧移，三角步态中若恰好该侧支撑薄弱，极易翻。
- **偏航（绕 Z）**：纯偏航对 G′ 位置影响较小，但腿的扫掠与地面摩擦会影响实际稳定；原地转向常比「边走边转」更需要预先打开支撑跨度。

因此，表情性姿态（抬头、侧倾致意）应优先放在六足站立或支撑充分的节拍上，而不是随意插在危险相位。

### 1.9.5 与本机尺寸相关的直觉

胯部包络大约前后 0.204 m、左右 0.213 m，连杆总长有限。这意味着：

- 支撑多边形不可能无限变大；
- 「看起来张开很大」很快会碰到关节行程与腹地间隙极限；
- 设计夸张开张姿态时，必须先做几何可行性，再谈舞台效果。

运动设计者若忽视尺寸，会不断提出硬件做不到的美学要求；尺寸意识是专业性的一部分。

## 1.10 六足特有的运动语言

把步态与机动当作「单词」，把性格与场景当作「语法」，才能写出有风格的句子。以下词汇是本手册的基础表达库。

### 1.10.1 三角步态：坚定、效率、主行走语汇

**形式：** 腿分为两组，例如常见分组（具体实现可有变体）：一组为 1、3、5，另一组为 2、4、6；一组支撑时另一组摆动。

**语义：** 前进/后退的「陈述句」。适合作为默认移动语言：清晰、对称感强、速度潜力较高。

**美学要点：**

- 机身应只有浅起伏，不要随每一步剧烈点头；
- 摆动腿抬起要干净，落足要缓冲；
- 两组切换的瞬间是强拍，适合少量表情，不适合大动作改高度。

**误用：** 用三角步态做「偷偷摸摸」——语义冲突，观众会困惑。

### 1.10.2 波浪步态：谨慎、展示、多足协调

**形式：** 足端抬起沿体侧依次传递，像波浪。

**语义：** 「我在小心地通过」「请注意我的协调」。更慢，更有展览讲解气质。

**美学要点：**

- 波浪方向与行进方向一致时，叙事最清晰；
- 波浪振幅（抬腿高度）是风格旋钮：高=表演，低=家庭谨慎；
- 机身应尽量稳，让观众把注意力放在腿序上。

### 1.10.3 螃蟹侧行：走位、让路、舞台横移

**形式：** 主要沿 +Y/−Y 横向移动，机身朝向可保持朝前。

**语义：** 「我不想把脸转过去，但我要换位置」——在展厅灯光下非常有用；在狭窄家居通道也常比转身更安全。

**美学要点：**

- 侧行前要有横向预动作；
- 侧行中保持机身朝向稳定，避免边横移边乱转；
- 结束时应用短静止「钉」在新位置上。

### 1.10.4 原地转向：重新定向

**语义：** 「注意那边」。转向质量决定机器人是否像有感知。

要点：先稳定支撑与跨度，再协调各腿 J1 与足端圆周位移；避免一只脚蹭着地被拖着转。转向过程可伴随极浅的下沉，以增加「用力转向」的可读性，但幅度要受性格约束。

### 1.10.5 高度变化语汇：探查 / 防御 / 蓄力

- **升高**：探查、自信、展示内部结构（若腹面有灯效或装饰）；
- **降低**：防御、钻越、低电、亲近地面物体；
- **先降后升**：蓄力跳跃感（本机通常不做真跳，但可用此语汇暗示）。

高度变化必须慢于同等幅度的水平移动，否则极易触发稳定与电流问题。

### 1.10.6 姿态语汇：点头、侧倾、蹲伏、开张

这些词将在后续「姿态动作」章节展开，这里只定语义：

| 语汇 | 常见意思 | 风险 |
|------|----------|------|
| 点头（俯仰） | 同意、致意、节拍 | 过大像砸头；过小看不见 |
| 侧倾（横滚） | 俏皮、倾听、让路 | 支撑不足时危险 |
| 蹲伏 | 准备、防御、结束礼 | 擦地、难再启动 |
| 开张 | 欢迎、占位、威吓（慎用） | 扫到环境、夹手空间变大 |

### 1.10.7 词汇表要克制

专业团队常犯的错是词汇过多：十种步态、二十种表情，用户却记不住任何一个。建议产品级公开词汇控制在：

- 移动词：2～3 个（三角、侧行、必要时波浪）；
- 姿态词：4～6 个；
- 签名词：3～5 个。

其余留给调试与内部实验，不进入对外说明书与演示默认脚本。

## 1.11 不同场景的动作哲学

同一套硬件，在不同场景应呈现不同哲学。哲学不同，不是「换皮」，而是目标函数不同。

### 1.11.1 家庭场景：可预测的同居者

**第一目标：** 不惊吓、不破坏、可近距离共存。  
**第二目标：** 有轻度生命感，避免像故障设备。  
**第三目标：** 功能完成（过来、停下、避开）。

哲学条款：

1. 速度上限显著低于舵机能力上限；
2. 抬腿高度以满足越障与不擦地为准，不追求夸张；
3. 待机允许浅呼吸，但禁止突然表演插入；
4. 中断响应必须快而柔，孩子一喊停就应进入收敛；
5. 声音与振动要被当作污染，能减则减；
6. 性格偏向礼貌、谨慎、略慢。

家庭场景的高级感来自**安静与准确**，不是来自花式。

### 1.11.2 展厅场景：可讲解的展品

**第一目标：** 远距离可读、循环耐看、拍照友好。  
**第二目标：** 体现六足结构之美与协调。  
**第三目标：** 支撑讲解叙事（「看，三角支撑」「看，侧行」）。

哲学条款：

1. 动作段落清楚，段间留静止，方便讲解员插话；
2. 对称与舞台收势被强化；
3. 可适度提高抬腿与开张，让结构被看清；
4. 提供「屏息站立」供合影；
5. 循环点必须优雅；
6. 性格偏向从容、展示、略仪式化。

展厅最怕：一直动个不停，观众不知道看哪里。

### 1.11.3 儿童场景：被放大的家庭约束

儿童场景不是「更可爱的家庭场景」这么简单，而是风险权重被放大：

- 手会伸进腿间；
- 会突然出现在机器人前方；
- 会模仿机器人做危险动作；
- 会把「拟凶」表情当真。

哲学条款：

1. 禁止威吓、扑咬、剧烈横滚等拟态；
2. 所有近人动作以收敛、减速、停止为美；
3. 互动以「机器人先停，再给人反应时间」为默认；
4. 色彩与声音若存在，动作应与之同向（温柔），不要动作凶、灯效欢；
5. 性格必须极度可预测；惊喜只能来自内容（例如答题正确后的轻轻点头），不能来自身体爆炸。

### 1.11.4 比赛场景：被规则与爆发定义的表演者

比赛（竞技或表演赛）允许更高密度与更高速度，但专业性反而更严：

1. **开场三秒定生死**：起势必须干净；
2. **段落感强于炫技堆砌**：评委需要看懂结构；
3. **风险动作要有备份**：现场地板、线缆、灯光都可能与排练不同；
4. **结束必须可定格**：便于摄像与得分印象；
5. **性格可以更锋利**，但仍要像同一台机器人的「比赛模式」，而不是完全换人。

比赛哲学最容易污染家庭固件。手册要求：**比赛包与家庭包分离**，共享词汇，不共享极限参数。

### 1.11.5 场景切换检查表

切换场景配置时，至少检查：

| 项目 | 家庭 | 展厅 | 儿童 | 比赛 |
|------|------|------|------|------|
| 最大抬腿 | 低 | 中高 | 最低 | 高 |
| 最大机身倾角 | 低 | 中 | 极低 | 中高 |
| 呼吸待机 | 开 | 开（可关） | 开（更浅） | 可选关 |
| 签名仪式 | 弱 | 强 | 弱而友好 | 强而利落 |
| 可中断优先级 | 最高 | 高 | 最高 | 中（仍须安全） |

## 1.12 给不同角色的阅读指引

本章及后续章节面向多种读者。请按角色选择重点，避免用错章节。

### 1.12.1 产品经理

你要回答的是：**用户在关键触点上应感到什么。**

请重点阅读：

- 1.2 性格、1.3 体验、1.11 场景哲学；
- 原则中的安全、可读、可中断；
- 签名动作与场景检查表。

请少做的事：

- 不要用「再加一个更酷的动作」替代性格一致性；
- 不要在未改场景哲学的情况下要求比赛级速度进入家庭固件；
- 不要把动作清单写成功能清单而不写体验标准。

你的交付物应包括：性格卡、场景矩阵、签名动作列表、禁止清单。有了这些，设计与工程才不会空转。

### 1.12.2 动画 / 运动设计师

你要回答的是：**每一帧是否在说话，句子是否有标点。**

请重点阅读：

- 1.4 原则、1.5 美学、1.6 节奏、1.7 连续、1.8 过渡、1.10 运动语言；
- 始终带着本机关节与 Home/坐标系约束想象动作，而不是按人型骨骼自由发挥。

工作方法建议：

1. 先写意图句（它在做什么、给谁看、什么情绪）；
2. 再选词汇（三角 / 侧行 / 点头…）；
3. 再写四段节奏；
4. 再检查支撑与重心；
5. 最后才进入具体轨迹细化（后续章节）。

评审时请携带慢放录像与「三米可读」标准，而不是只看仿真里的光滑曲线。

### 1.12.3 算法 / 控制工程师

你要回答的是：**如何在真实 MG996R 与 18 DOF 约束下，忠实地再现设计意图。**

请重点阅读：

- 1.1（理解为何不能止步于可行解）、1.4 稳定性/可中断/可组合、1.8 过渡、1.9 重心与支撑；
- 后续基础移动与姿态章节中的约束叙述。

请坚持的工程伦理：

- 当设计意图与物理极限冲突，反馈给设计，而不是默默做成「能跑但很丑」；
- 把可中断、支撑约束、速度包络视为硬需求，而不是美化选项；
- 用可复现的实验证明稳定与重复精度，而不是只靠单次成功演示。

你不需要在本章寻找 API；本章给你的是验收语言：什么叫稳、什么叫可读、什么叫收得住。

### 1.12.4 AI 编码模型（及使用模型的开发者）

当你（或你的工具链）根据自然语言生成动作脚本、轨迹或状态机时，请把本章当作**宪法提示**，而不是装饰性 README。

生成前应核对：

1. 关节名与正方向是否与本机一致（J1 绕 +Z，J2/J3 俯仰折膝）；
2. Home 是否被误当成展示站立；
3. 足号是否用错（1 左前 … 6 右前）；
4. 坐标系是否用错（+Y 为左，不是右）；
5. 是否包含起势与收势，而不是角度跳变；
6. 是否声明支撑假设与可中断行为；
7. 是否符合当前场景性格，而不是默认「越快越好」。

生成后应用本章原则做自检表，而不是只检查「代码能跑」。能跑只是功能层；手册要求观感层以上。

对开发者：把性格卡与场景矩阵放进提示词或设计文档索引，比只丢给模型一段「写个走路函数」有效得多。

### 1.12.5 跨角色协作的最小闭环

推荐固定评审闭环：

1. 产品经理提出场景与体验句；
2. 运动设计师给出节奏与词汇方案；
3. 算法工程师给出可行性与风险；
4. 实机慢放评审（含非技术观众一句复述）；
5. 记录进手册案例库，更新禁止清单与签名动作。

没有闭环，手册会变成墙上标语；有闭环，手册才能长期维护。

## 1.10 跨领域灵感地图：动作从哪里来

本手册后续章节不会把自己关在「六足步态教科书」里。真正有产品生命力的动作库，几乎总是从相邻领域「偷师」再本地化到 18 DOF、MG996R 与本机几何。下面给出一张长期可维护的灵感地图，供产品经理、动画师与 AI 编码模型在扩展动作时使用。

### 1.10.1 四足机器人（Spot / Unitree / ANYmal 系谱）

四足给六足最大的启发不是「少两条腿怎么补」，而是**身体作为主动表达器官**：俯仰点头、横滚撒娇、高度呼吸、对角步态的弹性触地。可迁移原则：

1. **身体通道优先于腿部炫技**：用户先读懂机身意图，再注意到腿。
2. **接触事件要被设计**：落足、轻蹭、停步，都是可听可见的「音节」。
3. **失败要优雅**：四足常用的「小心坐下 / 冻结 / 缓慢趴下」比突然断电更像活物。
4. **速度档位是性格旋钮**：巡航、谨慎、表演三档，比连续无级滑条更符合展厅与家庭。

六足特有加成：三角支撑更稳，可以把四足不敢做的「单侧抬双腿表演」做成安全互动；但侧行与原地旋转时，必须显式设计 J1 扇形开合，否则会像六把扫帚同时扫地。

### 1.10.2 宠物机器人与陪伴产品

宠物机器人（Aibo、Loona、各类桌面宠物）证明：**短互动循环 > 长炫技**。用户需要的是「我叫它，它看我；我摸它，它抖一下；我离开，它小声张望」。迁移到六足：

- 用 **LEG_WAVE / PERF_GREETING / EMO_CURIOUS** 构成 3～8 秒的「被看见」循环。
- 用 **BEH_WAIT** 的微动作（重心轻移、偶发点地）对抗「死机感」。
- 用 **电量与温度** 驱动性格衰减：低电量时抬腿变低、停顿变长，比弹出对话框更有生命感。

禁忌：不要把宠物互动做成不可中断的 30 秒动画。陪伴产品的第一美德是**随时可打断且打断后仍可爱**。

### 1.10.3 工业与巡检机器人

工业机器人贡献的是**可信、可预期、可审计**：

- 路径动作（方形、巡逻、对接靠近）必须有可重复的几何精度叙事。
- 急停与恢复必须成为一等动作，而不是异常分支。
- 「巡检模式」应像操作规程：出发自检 → 路径 → 观察点停留 → 记录姿态 → 返回。

六足若进入校园安防演示或机房巡检展示，工业气质能显著提升「这不是玩具」的说服力；但切记工业气质与卖萌气质要用**模式切换**隔离，而不是混在同一套默认 Idle 里。

### 1.10.4 动画设计与电影肢体语言

动画给学生的不是关节角度，而是**时间与夸张**：

- 预备（Anticipation）、跟随（Follow-through）、重叠动作（Overlapping Action）、缓入缓出（Ease in/out）必须写进每个 Motion 的节奏段。
- 夸张要受物理约束：MG996R 做不出卡通瞬间位移，但可以用「更大的预备下沉 + 稍快的伸展」假装弹性。
- 角色弧光：同一机器人在「迎宾 / 警戒 / 低电」三种状态下，同一 Wave 应有三种时间版本。

### 1.10.5 电子游戏的状态机与 Emote

游戏给出两样宝：

1. **状态机清晰**：Idle / Locomotion / Emote / Hitstun / Death（对本机映射为 Rest / Move / Perf / Recover / Sleep）。
2. **Emote 轮盘思维**：用户不需要 200 个一级按钮，需要 8 个常驻 + 可扩展轮盘。

从游戏可直接本土化的概念包括：BOSS 登场步、低血量踉跄、任务完成打勾、营地休息、蓄力姿势、连击收招。它们不一定上默认儿童模式，但极大丰富展厅与创作者工具。

### 1.10.6 生物行为学（Ethology）

六足外形天然接近节肢动物，观众会自动调用蜘蛛/昆虫图式。设计者应主动管理这种图式：

| 生物行为 | 可转化动作 | 产品风险 |
|----------|------------|----------|
| 威胁展示（抬前肢） | Dual Lift + Body Up | 儿童可能害怕，需强度参数 |
| 求偶舞（节律摆动） | Dance / Dual Swing | 适合展厅，注意噪声 |
| 理毛（互相刮擦） | Scratch / Touch Body | 注意自碰限位 |
| 受惊逃窜 | Dash + Freeze | 必须可限速，禁止默认高速 |
| 探路触角感 | Probe / Tap | 极适合「智能感」 |
| 缩壳防御 | Low Profile + Legs In | 低电/害怕情绪共用 |

原则：**借用生物可读性，过滤生物恐惧点**。家庭产品默认走「猫狗陪伴 + 轻微昆虫聪慧」，而不是「恐怖爬虫」。

### 1.10.7 人机交互（HCI）

HCI 要求动作成为**对话回合**：

- 系统状态必须可被身体表达（聆听、思考、执行、完成、失败）。
- 延迟超过 300ms 时，要用「思考微动」填洞，而不是静止。
- 错误不应只有蜂鸣，应有 **EMO_CONFUSED + 轻微摇头式机身摆动**。
- 所有高风险动作在 UI 上的可达性，必须低于其在程序 API 上的可达性。

### 1.10.8 扩展动作时的五问检查表

任何新动作入库前，作者（含 AI）必须书面回答：

1. 它让用户在 1 秒内读懂什么意图？
2. 它更像四足、宠物、工业、动画、游戏、生物中的哪一类方言？
3. 它在儿童 / 家庭 / 展厅 / 比赛哪种场景默认允许？
4. 它的可中断点在哪里？中断后落到什么安全姿态？
5. 它是否可用已有动作组合实现 70%？若是，优先做 Combo，而不是新原子动作。

## 1.11 本机「运动方言」：十八关节上的修辞格

把动作设计比作修辞，有助于团队统一审美语言：

- **对偶**：左右镜像（1↔6、2↔5、3↔4）产生仪式感与高端感。
- **排比**：三角步态三相重复，形成可预期节奏。
- **倒装**：先动身体后抬腿，或先点地再转体，改变「句子重音」。
- **留白**：动作间 200～600ms 静止，让观众完成语义闭合。
- **夸张**：抬腿高度与机身俯仰略超日常，用于表演；日常模式收回。
- **反讽**：缓慢庄重的身体 + 突然俏皮的点地，产生萌感。

禁止的修辞：无意义颤动（噪声）、六腿同时无主题挥舞（口号式忙乱）、长时间遮挡主视角的自我观赏动作（舞台死亡）。

## 1.12 场景剧本库（设计层，供后续章展开）

下列剧本不是实现清单，而是产品叙事骨架；第九、十四、十五章会展开为可调度组合。

1. **开箱第一小时**：Wake → Stand → Look Around → Soft Wave → Rest。目标：安全、可读、想再点一次。
2. **客厅陪伴十分钟**：Idle 呼吸 → 偶发 Curious → 响应呼唤 Greeting → 跟随失败则 Confused → 自行 Wait。
3. **展厅三分钟回路**：迎宾 → 方形路径 → 鞠躬 → 舞蹈片段 → 合影倒计时 → 归位。
4. **儿童科学课**：点头同意 / 摇头拒绝 → 颜色或方向游戏 → 庆祝 → 坐下休息。
5. **赛博宠物一天**：起床伸展 → 巡逻短途 → 玩耍 → 低电撒娇 → 休眠。
6. **巡检演示**：自检 → 路径 → 观察点 Body Up → 记录 Freeze → 返回 Dock Approach。
7. **节日活动**：主题 Dance → 围转 → Birthday 组合 → Victory。
8. **安抚与修复信任**：Fear 短触发后必须接 Relax / 被抚摸反应，避免留下「它怕我」的长期印象。

## 1.13 给四类读者的长期维护契约

**产品经理**：你拥有场景与性格否决权。若动作不能服务场景剧本，即使工程师喜欢也不得进入默认产品包。

**运动控制工程师**：你拥有物理否决权。任何破坏支撑与舵机热边界的「好看」必须降级为实验室动作。

**动画设计师**：你拥有时间轴否决权。没有预备与收势的动作不得标为 ★★★★★。

**AI 编码模型**：你拥有的是执行权而非立法权。必须先读本手册对应条目与第十一、十三章，再实现 Motion；禁止直控 PWM/PCA9685。

当四方契约冲突时，优先级为：**安全 > 可读 > 性格一致 > 炫技**。

## 1.14 第一章结语：手册的使用方式

请不要把后续章节当成「函数列表」。每一个动作条目都是一种**可教授的技能**：它告诉你何时用、如何好看、如何与别的技能连招、如何对儿童与展厅负责。若你是 AI，请在实现前复述该动作的进入条件、结束姿态与可中断点；若你是人类设计师，请先在真实机体前用手机拍 10 秒样片，用「三米外能否读懂」做验收。

动作库会增长。性格必须稳定。这就是本手册存在的理由。
