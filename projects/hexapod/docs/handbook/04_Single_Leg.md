# 第四章 单腿动作

本机：蜘蛛六足18DOF；J1 Hip Yaw / J2 Hip Pitch / J3 Knee Pitch；MG996R；Home J1=90° J2=180° J3=125°；+X前 +Y左 +Z上；腿1左前 2左中 3左后 4右后 5右中 6右前。

单腿动作是六足的手势语言。执行前必须 Shift 移心，五腿支撑。灵感：宠物抬爪、人类手势、昆虫触角、音乐鼓点、教学指示。

### 挥手（Wave）

#### 动作名称
- **英文**：Wave
- **中文**：挥手
- **MotionID**：`LEG_WAVE`
- **推荐等级**：★★★★★
- **适用场景**：迎宾儿童视频通话
- **动作目的**：指定腿做可读挥手

#### 动作描述

宠物抬爪与人类挥手的六足版。先Shift移心，再抬腿以J1/J2画弧，默认腿1或6。这是第一社交技能。 设计时请同时考虑支撑多边形、MG996R速度包络、可中断点与结束姿态可读性。它应当能被产品经理用一句话讲清意图，也能被动画师画出时间轴，更能被AI按Motion合同实现。

跨领域来源与本地化：社交emote。本机：蜘蛛六足18DOF；J1 Hip Yaw / J2 Hip Pitch / J3 Knee Pitch；MG996R；Home J1=90° J2=180° J3=125°；+X前 +Y左 +Z上；腿1左前 2左中 3左后 4右后 5右中 6右前。
本动作必须通过 Motion 层调度，支持 `speed`、中断、暂停与恢复；禁止直接控制 PWM 或 PCA9685。实现前请阅读本条目的进入/退出条件与注意事项，并在实机用「三米外能否读懂」做验收。

#### 视觉效果

清晰的你好

在家庭近距观看时，细节与声音（落足、舵机音色）同样构成体验；在展厅 throng 中，机身高度、对称性与节奏句读比足端微轨迹更重要。若观众只能看出「腿在抖」，则本动作尚未达到入库标准。

#### 动作流程

移心→抬腿→挥2–4次→放回

流程中的每一步都应能被上层组合动作引用为阶段名（例如 `prep` / `main` / `hold` / `recover`），以便第九章剧本与APP时间轴对齐。

#### 动作节奏

欢快

统一节奏语法为「预备—主动作—延持—收势」。家庭与儿童模式加长预备与收势；比赛与舞台可压缩延持，但不得删除收势。建议在延持段设置默认可中断点；急停则随时抢占。

#### 动作参数

leg_id,cycles,amp

#### 推荐默认参数

leg_id=1,cycles=3

#### 速度建议

中

MG996R 空载约 0.14–0.17 s/60°。默认档必须可连续重复而不明显啸叫、不撞地；高速档仅用于表演并计入热管理与冷却。

#### 腿部动作

#### 推荐步态
