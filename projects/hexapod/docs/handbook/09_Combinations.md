# 第九章 组合动作

组合动作是产品的「剧本」。单动作是词汇：站立、挥手、鞠躬、跳舞、警戒；组合动作是句子与段落：它规定先说什么、后说什么、哪里停顿、哪里高潮、何处留给相机。观众记住的往往不是某个关节角，而是一段完整情节——进门被欢迎、胜利仪式、睡前温柔、展台循环短剧、第一次见面的郑重、低电安慰、打翻东西后的道歉、生日祝福、赛后颁奖、雨天避让、主人回家冲刺欢迎，以及安静图书馆模式的克制切换。

组合设计硬规则：衔接必须经过可站稳中间态；抬腿表演前后锁定支撑多边形；高潮动作有冷却与净空；任何组合可被急停并落到安全站立；性格一致，不在同一剧本里忽萌忽恐怖；儿童与展厅模式可降级强度。组合层向下调用移动、姿态、单腿双腿、表演与情绪，向上被行为模式与 APP 节目单调用。本章不写驱动实现，只规定运动叙事与参数语义。

## Greeting 组合：Stand→Wave→Bow→Stand

**名称**：问候组合（Greeting Sequence）

**MotionID**：COMBO_GREETING

**星级**：★★★★★

**场景**：门口迎宾、来客亮相、演示开场、语音「打个招呼」、展厅讲解引入、家庭访客第一互动。凡需在短时间内建立礼貌印象的社交入口均可调用。

**目的**：用最短礼仪链完成可见欢迎：站稳、挥手致意、鞠躬示敬、再站回可交互姿态，形成可拍照、可打断、可复述的标准社交句子。

**描述**：问候组合把站立、挥手、鞠躬串成有起承转合的句子，而不是同时乱动。先确保对称站立可读且朝向访客，再抬前腿挥手建立亲切，然后全身前俯鞠躬建立礼貌，最后回到站立等待。挥手时支撑多边形必须预先锁好：用腿1挥手则强化腿2、3、5、6支撑；用腿6则镜像。鞠躬阶段六腿协同改变等效腿长使机身俯仰前倾，足端尽量不滑。整段像人类接待：看着对方、打招呼、欠身、直起。性格为温和蜘蛛管家，不突然弹跳、不侧向威吓。访客过近时应先由迎宾行为后退半步再进入。儿童降低鞠躬深度并加大社交距离。熟客可缩短鞠躬停顿，只保留轻挥与浅礼。衔接处每个箭头都经过可站稳中间态，避免挥手未收就鞠躬造成前倾叠加失稳。

**视觉效果**：三到八秒完成「它会打招呼」：稳定轮廓、清晰挥手弧线、可读前倾、回正站立。社交峰值，易被短视频截取。

**流程**：Stand 对称站稳朝向访客 → 支撑锁定半秒 → Wave（腿1或腿6，二至四次）→ Bow（中等前俯并停顿）→ Stand 起身回亲切站立。串联：Stand→Wave→Bow→Stand。

**节奏**：站稳半秒到一秒；挥手轻快；鞠躬偏慢；起身中速。总长四到十秒。商务拉长鞠躬，萌宠加密挥手。

**参数**：wave_leg；wave_count；bow_depth；hold_bow；face_target；politeness；child_safe；approach_clearance。

**默认参数**：wave_leg=自动外侧前腿；wave_count=3；bow_depth=中；hold_bow=0.6s；face_target=开；politeness=中。

**速度**：挥手中速；鞠躬慢；起身中速。儿童全局降速约三成。

**身体姿态**：亲切高站；挥手时对侧略补高；鞠躬前倾八到十八度；结束水平亲切站立。

**腿部动作**：挥手腿 J1/J2/J3 画弧；支撑腿锁定；鞠躬六腿改等效长度，足不挪。触地高度低于 Home 悬空标定。

**步态**：组合内无移动。靠近由迎宾行为在组合前完成。

**可组合**：被 BEH_GREETER 调用；可接致谢离开、第一次见面仪式；可换害羞点头版；可作展厅开场第一句。

**结束姿态**：面向访客的对称站立，机身水平，六腿均匀承重。

**进入退出条件**：进入：已站稳且鞠躬净空足够。退出：播完；急停站稳；冷却防刷；低电降级浅点头。

**变种**：萌宠双点地；商务深鞠躬少挥；静音点头；窄通道单侧挥。

**扩展**：熟客减鞠躬；多访客只挥不鞠；语音「你好」直触；与拍照姿势衔接。

**APP语音自动**：APP「打招呼」。语音：打个招呼、欢迎、说你好。自动：迎宾人体触发。

**第一印象**：它好有礼貌！第一次演示几乎必用。

**值得实现理由**：最低成本最高传播的礼仪链；验证支撑锁定与姿态衔接。

**难度**：低到中。难在无晃衔接与左右镜像一致。

**实现顺序**：Stand/Wave/Bow 验收后立即编排；再加礼貌参数与儿童降级。

**注意事项**：挥手前锁支撑；鞠躬防滑防撞膝；儿童加大距离；勿与快退重叠；斜坡不做深鞠躬。 禁止边走边大幅抬双侧前腿；急停优先对称站稳。

## Celebrate 组合：Jump→Wave→Dance→Celebrate

**名称**：庆祝组合（Celebrate Sequence）

**MotionID**：COMBO_CELEBRATE

**星级**：★★★★★

**场景**：任务成功、比赛得分、生日高潮、展厅尖叫点、语音「庆祝一下」、通关与打卡成功反馈。

**目的**：用递进高潮表达喜悦：弹跳释放、挥手招呼、舞蹈展开、定格庆祝，让情绪从爆发走到可拍照收束。

**描述**：庆祝组合是情绪升级剧本。跳跃或欢快弹跳打开兴奋，挥手把情绪递给观众，短舞给出时长与变化，Celebrate 定格留下纪念帧。净空与附着力不足时必须用 Happy Bounce 替代真跳，并在参数声明。节奏由爆发到展开再到收束，避免持续高负荷导致舵机过热。舞蹈以机身俯仰滚转与交替点地为主，禁止旋转失控。腿1与腿6可作表演前肢，腿2与腿5维持支撑关键，腿3与腿4稳住后域。儿童默认禁跳；地毯与滑地分参。定格够拍照但不可久堵转。性格保持欢快，不突然切愤怒或惊吓。若前序已是高强度动作，应插入站稳冷却再进入本组合。音乐同步为可选项，节拍丢失时回退内部节拍器。

**视觉效果**：「起—扬—舞—定」四段清晰；结尾定格适合封面与尖叫点。

**流程**：Jump/Happy Bounce → Wave（单或双前腿快挥）→ Dance 短版八到十五秒 → Celebrate 定格两到三秒。无跳版从 Bounce 或 Wave 起。

**节奏**：跳短促；挥手密；舞有拍；定格停稳。总长十五到三十秒。高潮后必减速。

**参数**：jump_enable；dance_style；celebrate_pose；intensity；photo_hold；music_sync；thermal_guard。

**默认参数**：jump_enable=按场地；dance_style=欢快短版；photo_hold=2.5s；intensity=中高；thermal_guard=开。

**速度**：前半偏快；定格前减速。儿童与近距展厅再降档。

**身体姿态**：低蓄力到高开，舞中起伏，结尾挺拔；横滚受控以免误读翻倒。

**腿部动作**：弹跳同步蓄力释放；舞段交替点地；定格可抬前腿但三角支撑牢固；J3 勿久贴折叠极限。

**步态**：弹跳特殊相位；舞可用节奏步；定格无步态。

**可组合**：胜利仪式可调用；生日可嵌本段；Demo 高潮；赛后颁奖可接其后半。

**结束姿态**：Celebrate 定格或兴奋站立，六腿回对称。

**进入退出条件**：进入：成功事件且场地允许。退出：播完/急停站稳；冷却；过热降级无跳短舞。

**变种**：无跳；静音点地；儿童超慢；仅定格拍照。

**扩展**：节拍对齐；欢呼加强则加时长；与颁奖衔接；分享触发加长定格。

**APP语音自动**：庆祝按钮。语音：庆祝一下、太棒了、耶。自动：任务成功回调。

**第一印象**：好开心！情绪感染强。

**值得实现理由**：高潮记忆点；验证弹跳安全与热保护；利于传播。

**难度**：中高。安全边界与儿童降级是关键。

**实现顺序**：先无跳 Wave-Dance-Celebrate，再评估 Jump，最后音乐同步。

**注意事项**：净空防滑；禁儿童跳；定格勿久堵；斜坡禁用跳；急停站稳。 禁止边走边大幅抬双侧前腿；急停优先对称站稳。

## Wake Routine 唤醒例程

**名称**：唤醒例程（Wake Routine）

**MotionID**：COMBO_WAKE_ROUTINE

**星级**：★★★★★

**场景**：早晨、开馆、语音起床、触摸唤醒后的完整醒来表演、日程开机仪式。

**目的**：把休眠到清醒戏剧化为可分享的起床剧本，含撑起、伸懒腰与短问候，避免工业设备上电感。

**描述**：唤醒例程从休眠低姿开始：微呼吸动→对称撑起至半高→伸懒腰（前腿交替上举或俯仰拉伸）→可选轻震身→可选小挥手→清醒站立等待。它比单纯抬升多了生活气。与行为 BEH_WAKE 共用阶段语义，组合更强调表演完整性与拍照点。安静变种去掉震身与挥手，适合夜间被吵醒或图书馆模式结束后的克制醒。撑起过程必须保持六腿接触顺序可读，禁止单侧猛撑导致横滚。伸懒腰每次只解放一条摆动腿，其余五腿或三角支撑锁死。震身幅度要小，仅传达「抖掉睡意」。结尾站立应略低于表演英雄姿，保留刚醒的柔软感，再交给迎宾或等待行为。工作日可偏利落，周末可偏慵懒，由 energy 参数调节升起时间与拉伸次数。

**视觉效果**：宠物醒来全流程：从团缩到舒展再到点头致意，观众会觉得「好可爱」。

**流程**：Sleep/低姿微呼吸 → 撑起半高 → Stretch 伸懒腰 → 可选 Shake Body → 可选小 Wave → Stand 清醒站立。

**节奏**：前慢后明快；总六到十八秒。早晨偏慢，开馆可稍利落。

**参数**：stretch_style；shake_enable；wave_enable；rise_time；energy；quiet_mode。

**默认参数**：stretch_style=交替前腿；shake_enable=开；wave_enable=场景化；rise_time=10s；energy=中。

**速度**：升起慢；拉伸中慢；可选挥手中速。安静模式全局更慢。

**身体姿态**：低→半高→拉伸超控→回标准清醒站立，俯仰从微负到零。

**腿部动作**：对称撑起；拉伸每次一腿；震身小幅；挥手锁支撑。J2/J3 渐进改变等效腿长。

**步态**：无前进。原地微调落点允许极小步。

**可组合**：被 BEH_WAKE 调用；其后接等待、迎宾、陪伴；可接第一次见面或问候组合。

**结束姿态**：清醒对称站立，面向默认交互方向。

**进入退出条件**：进入：睡姿稳定且收到唤醒事件。退出：完成移交；失稳则恢复站立；低电限制拉伸幅度。

**变种**：慵懒；安静；战斗醒（少萌多警戒）；展厅开馆利落版。

**扩展**：按唤醒源改结尾；工作日/周末不同能量；与日程灯光联动。

**APP语音自动**：起床按钮。语音：起床了、早安。自动：日程闹钟。

**第一印象**：有生活仪式的机器人，不像冷冰冰开机。

**值得实现理由**：日周期情感与开场广告；休眠体系的表演闭环。

**难度**：中。难在慢速撑起不抖、拉伸不失稳。

**实现顺序**：随休眠唤醒行为一起做；先无震身版，再加可选表演。

**注意事项**：未站稳不移动；夜间用安静变种；防侧翻；拉伸禁双侧前腿同时高举。 禁止边走边大幅抬双侧前腿；急停优先对称站稳。

## Sleep Routine 睡眠例程

**名称**：睡眠例程（Sleep Routine）

**MotionID**：COMBO_SLEEP_ROUTINE

**星级**：★★★★★

**场景**：晚安、打烊、低电前优雅下线、家长说睡觉、展厅闭馆仪式。

**目的**：用可理解的睡前仪式进入安全低姿，而不是突然塌软，给空间「安静下来」的信号。

**描述**：睡眠例程：停止当前活动并站稳→可选哈欠式俯仰伸展→转向休眠朝向或「窝」方位→可选极小绕步或原地落点微调→分阶段降低机身→收腿定格 Sleep。可插入短暂找窝点地，增加宠物感。与 BEH_HIBERNATE 对仗，组合侧重仪式完整与温柔。展厅打烊可与灯光同步，做成正式关灯版。降段是安全关键：速度必须明确慢，六腿压力均匀，禁止一条腿先抽空。收腿时保持支撑多边形逐步缩小但仍稳定，最终低姿重心投影仍在足端凸包内。若电量危急，可跳过绕步与哈欠，走快速安全躺下变种，但仍禁止自由落体式塌软。结束后只允许唤醒例程或紧急恢复拉起，不接受庆祝等高能组合直切。

**视觉效果**：说晚安的对象；空间节奏明显变慢，适合家庭晚间与闭馆镜头。

**流程**：Stand 收束 → 可选 Stretch/哈欠 → 转向休眠朝向 → 慢降机身 → 收腿 Sleep → 定格微呼吸或静止。

**节奏**：全程慢，降段最慢；八到二十二秒。危急低电可压缩到五到八秒但仍分阶段。

**参数**：nest_yaw；stretch_before；descent_time；circle_before；final_breath；fast_safe。

**默认参数**：stretch_before=开；descent_time=12s；circle_before=关；final_breath=极低或关。

**速度**：明确慢。任何加速请求在本组合内应被忽略或仅微调。

**身体姿态**：高到低逐步团缩；俯仰可先微上扬再低下，模拟打哈欠。

**腿部动作**：对称收；足不滑；可先微调落点再降；最终低姿膝关节折叠受软限位约束。

**步态**：最多一小段调整步；禁止奔跑后直接睡。

**可组合**：被休眠行为调用；其后只能唤醒例程拉起；低电安慰组合可在其前播放。

**结束姿态**：稳定 Sleep 低姿，面向 nest_yaw。

**进入退出条件**：进入：可站稳且无禁睡标志。退出：完成定格；急停则改为安全低姿或站稳策略按电量选择。

**变种**：正式打烊；快速安全躺；儿童故事版加长哈欠；图书馆极静版无点地。

**扩展**：与窝点地图结合；与环境光联动；多机编队依次睡。

**APP语音自动**：睡觉按钮。语音：晚安、去睡觉。自动：打烊日程、危急低电前优雅下线。

**第一印象**：会被好好放下，而不是断电摔倒。

**值得实现理由**：安全与情感双重收益；闭馆与家庭刚需。

**难度**：中。慢降均匀与最终稳定最难。

**实现顺序**：先分阶段下降，再加哈欠与找窝，最后日程联动。

**注意事项**：禁止突然塌软；滑地降速；儿童远离收腿范围；热保护可先 Rest 再睡。

## Showroom Loop 展厅循环

**名称**：展厅循环短剧（Showroom Loop）

**MotionID**：COMBO_SHOWROOM_LOOP

**星级**：★★★★★

**场景**：展台无人值守循环、科技馆全日演示、商场橱窗、发布会背景循环。

**目的**：在有限舞台上用可循环短剧持续吸引路过目光，同时控制发热、噪声与审美疲劳。

**描述**：展厅循环把问候、展示步态、短表演、定格拍照点与复位串成可无缝循环的节目单。典型结构：复位站立→短暂张望→短前进或原地转向展示腿部节奏→挥手或点头→短舞或自豪姿态→定格两秒→退回起点站立→空窗休息。空窗休息是长时运行的生命线，用于散热与让观众「重新看见」。性格保持展览馆讲解员式礼貌与自信，不做惊吓或愤怒。所有子动作必须 exhibit_safe，禁跳或仅在确认地胶与护栏时启用极弱弹跳。循环计数与热保护触发后应降级为姿态展示循环。舞台边界用软围栏参数约束，绝不走出地毯或展台。可与灯光和语音旁白时间轴对齐，但旁白丢失时动作仍自洽。观众靠近时可打断切入迎宾，离开后从复位点续播，而不是卡在半截舞蹈。

**视觉效果**：远看一直有事做，近看每三十到六十秒有一个高潮定格，空窗后重新吸引注意。

**流程**：Reset Stand → Look around → 短展示步/转向 → Wave 或 Nod → 短 Perf → Photo Hold → 退回复位 → Rest 空窗 → 循环。

**节奏**：表演段紧凑，空窗明确。单圈建议四十到九十秒，空窗占十五到三十秒。

**参数**：loop_count；rest_ratio；stage_bounds；highlight_pose；allow_interrupt_greet；thermal_degrade。

**默认参数**：rest_ratio=0.25；allow_interrupt_greet=开；thermal_degrade=开；highlight_pose=自豪站立。

**速度**：中慢展示速；定格静止；复位中速。儿童观展时段再降。

**身体姿态**：以可读高站与轻度俯仰变化为主，避免长时间极限姿态。

**腿部动作**：展示步清晰抬落；表演段锁支撑；复位步短而准。

**步态**：短三角步或原地踏步；禁止长距离巡逻式离开舞台。

**可组合**：可嵌 Greeting、Celebrate 短版、Photo Pose；打断后接 BEH_GREETER。

**结束姿态**：舞台原点对称站立，或 Sleep 打烊版由日程切换。

**进入退出条件**：进入：展厅模式且舞台标定完成。退出：打烊、过热、急停、手动接管。

**变种**：静音橱窗；互动打断增强；节日换皮节目单；夜间低亮度慢版。

**扩展**：多机错峰循环；与屏幕字幕同步；按客流调节空窗。

**APP语音自动**：APP 节目单播放/暂停。语音：开始表演、停。自动：展厅模式空闲超时。

**第一印象**：展台上的小演员，而不是反复抽搐的样机。

**值得实现理由**：商业演示刚需；长时稳定性与内容运营入口。

**难度**：中高。难在循环无缝、散热与可打断恢复。

**实现顺序**：先单圈手编，再循环与空窗，再打断续播，最后热降级。

**注意事项**：护栏与跌落；噪声时段；舵机热；勿无限高潮；急停可见可恢复。

**第一印象**：它会摆拍！分享欲立刻上来。

**实现顺序**：先无抬腿定格，再抬腿姿，再换姿与快门联动。

**流程**：Stand → Angry posture + 克制跺踏/横摆 → Freeze breath → Calm rise → Soft nod → Neutral Stand。

**节奏**：怒段短（二到四秒）→ 呼吸停顿一到二秒 → 平静段四到八秒。总长八到十五秒。

**参数**：anger_intensity；stomp_enable；calm_style；child_soften；face_audience。

**默认参数**：anger_intensity=低中；stomp_enable=场景化；calm_style=慢起+点头；child_soften=开。

**速度**：怒段偏快但幅度受限；平静段明确慢。

**身体姿态**：先压低后回升；横摆从大到小直至静止。

**腿部动作**：跺踏轻且少；支撑不丢；平静段六腿匀压。

**步态**：基本无移动；最多怒段原地踏。

**可组合**：戏剧节目单；可接感谢离开或安静模式；勿与迎宾直接串联致误读。

**结束姿态**：中性平静站立。

**进入退出条件**：进入：表演指令且模式允许。退出：播完；儿童靠近则立刻切平静；急停站稳。

**变种**：教育超克制；舞台加强；无跺踏仅姿态。

**扩展**：与语音讲述脚本同步；情绪条 UI 演示；多段情绪弧。

**APP语音自动**：APP 戏剧页。语音：生气再冷静。自动：默认关闭，需显式使能。

**第一印象**：它懂情绪转折，不像坏脾气机器。

**值得实现理由**：人格深度；教育场景素材；情绪状态机演示。

**难度**：中。难在克制愤怒仍可读、转折清晰。

**实现顺序**：先平静恢复链，再加低强度怒段，最后儿童软化。

**注意事项**：禁止对人发泄；儿童弱化；勿高砸地；表演标注；勿残留敌意姿态。

## Thank And Leave 致谢离开

**名称**：致谢离开组合（Thank And Leave）

**MotionID**：COMBO_THANK_AND_LEAVE

**星级**：★★★★★

**场景**：接待结束、演示收尾、访客出门、语音「说谢谢然后回去」、迎宾闭环。

**目的**：用礼貌收束关系：致谢→转向离开→短距离退场→可选回望，避免突然掉头走开的无礼感。

**描述**：致谢离开是社交闭环的句号。先面向对方点头或浅鞠躬致谢，再清晰转向离开方向，以中慢速短距离离开到停靠点，可选在中途停顿回望点头，最后在停靠点站稳或进入等待/休眠。它把「谢谢」与「我走了」变成可读叙事。转向必须先站稳再偏航，离开步态稳定，不在致谢未完成时挪步。回望要短，避免犹豫式来回造成挡路。若空间狭窄，可改为侧移让路版：致谢后横移让开通道再离开。儿童场景离开速度更慢，回望可改为轻轻挥手。与第一次见面仪式对仗：一个开场一个收尾。低电时可缩约为致谢点头+原地待机，不再行走。

**视觉效果**：有教养的退场：谢、转、走、可选回望，观众不会觉得被甩开。

**流程**：Face guest → Thanks（nod/bow）→ Turn to exit → Walk short → Optional look-back → Dock Stand。

**节奏**：致谢一到二秒；转向一到二秒；离开三到八秒；回望一秒。总长八到十五秒。

**参数**：thanks_style；exit_yaw；dock_distance；lookback；leave_speed；narrow_strafe。

**默认参数**：thanks_style=点头或浅鞠躬；dock_distance=0.8m；lookback=开；leave_speed=中慢。

**速度**：致谢静止；离开中慢；儿童更慢。

**身体姿态**：致谢前倾或点头；离开时回正；回望时偏航。

**腿部动作**：致谢锁足；转向 J1 协调；离开短三角步。

**步态**：短距离前进/侧移；禁跑。

**可组合**：迎宾结束调用；展厅讲解结束；可接睡眠例程；可接回窝。

**结束姿态**：停靠点对称站立。

**进入退出条件**：进入：接待结束事件。退出：抵达停靠；路径阻塞则原地致谢结束；急停。

**变种**：侧移让路；无回望商务；挥手再见萌宠；原地致谢低电。

**扩展**：门口人体消失才离开；多访客只致谢不离开；与灯光送别同步。

**APP语音自动**：APP「谢谢离开」。语音：说谢谢然后回去、再见。自动：迎宾会话超时结束。

**第一印象**：好有礼貌的收尾。

**值得实现理由**：社交产品完整感；迎宾系统必备闭环。

**难度**：低到中。难在转向自然与挡路处理。

**实现顺序**：先致谢+转向，再短离开，再回望与侧移变种。

**注意事项**：勿撞人；窄道侧移；儿童慢；离开路径跌落检测。

## Alert Then Patrol 警后巡逻

**名称**：警戒转巡逻组合（Alert Then Patrol）

**MotionID**：COMBO_ALERT_THEN_PATROL

**星级**：★★★★☆

**场景**：异响复查后恢复巡视、夜间模式、安防演示、语音「警惕一下然后巡逻」。

**目的**：先用警戒姿态表达「我注意到了」，再平滑降级为持续巡逻，避免长期卡在紧张态。

**描述**：警戒转巡逻处理刺激后的行为降级：进入低姿或高敏站姿，前腿微探，机身指向刺激方位，短时冻结观察；若无升级威胁，则缓慢抬升回巡航高度，转向巡逻路径，开始 BEH_PATROL 语义的循环。关键是「降级可见」：观众应看出从紧张到值班的变化，而不是突然放松。警戒段禁止冲刺撞人，只允许原地或极小步进调整朝向。若刺激再现，可回到警戒；若确认安全，进入巡逻并降低灵敏度冷却。与愤怒转平静不同，本组合偏任务态而非情绪教育。展厅安防演示很加分，但家庭夜间需控制噪声与亮度相关动作。结束后不一定退出，而可能把控制权交给巡逻行为长期挂起。

**视觉效果**：先绷紧观察，再松开脚步开始巡，像值班生物完成一次核查。

**流程**：Alert posture → Orient to stimulus → Hold observe → Soft rise → Align path → Enter patrol loop。

**节奏**：警戒三到八秒；过渡二到四秒；其后巡逻自循环。

**参数**：alert_hold；stimulus_yaw；patrol_path；retrigger_window；noise_level；night_mode。

**默认参数**：alert_hold=5s；retrigger_window=10s；night_mode=按场景；patrol_path=rect。

**速度**：警戒近静止；过渡慢；巡逻中慢。

**身体姿态**：警戒压低或高敏微前倾；过渡抬升；巡逻近水平。

**腿部动作**：警戒前腿探；巡逻恢复标准抬腿；过渡禁止大跳。

**步态**：警戒无步或微步；巡逻三角步。

**可组合**：由异响事件触发；可接观察/寻找；威胁升级可切更高优先级安全停止。

**结束姿态**：不强制结束；移交时为巡逻巡航站姿或路径上的稳定步态相位点。

**进入退出条件**：进入：刺激且模式允许。退出：用户取消、低电、打烊、急停；巡逻本身可长期运行。

**变种**：家庭静音警戒；展厅夸张演示；仅警戒不巡逻。

**扩展**：多传感器融合方位；与摄像头预置位联动；日志记录核查点。

**APP语音自动**：APP 安防页。语音：警惕一下然后巡逻。自动：异响规则引擎。

**第一印象**：它会核查，也会继续值班。

**值得实现理由**：安防叙事完整；行为降级范例。

**难度**：中。难在降级平滑与再触发逻辑。

**实现顺序**：先警戒保持，再过渡，再接巡逻行为。

**注意事项**：禁冲人；夜间降噪；避免频繁警戒抖；跌落边界。

## 第一次见面仪式

**名称**：第一次见面仪式（First Meeting Ceremony）

**MotionID**：COMBO_FIRST_MEETING

**星级**：★★★★★

**场景**：开箱后首次激活、新主人绑定成功、贵宾介绍、语音「正式认识一下」。比普通问候更郑重、更长、更难忘。

**目的**：把「我们正式认识了」做成仪式：注视、自介绍式表演、握手模拟或礼节、共同定格，建立关系记忆点。

**描述**：第一次见面仪式高于日常 Greeting。建议链：从稍远位置谨慎靠近到社交距离→停稳注视（俯仰对准人脸方位）→自介绍式短表演（如轻点地报数三下或缓慢展示六腿张开再收回）→握手模拟或对称前腿礼→浅鞠躬→并肩或正面合影定格→亲切站立等待命名或摸头。它应只在关系状态为「未正式见过」时自动建议，见过之后降级为普通问候，以免每次都盛大。靠近过程必须可中断后退。握手模拟调用互动扩展章动作，但本章规定编排位置与停顿。性格郑重温暖，像典礼官。儿童版去掉靠近，改为原地仪式。展厅贵宾版加长定格。仪式中避免欢快大跳，把高潮留给定格与礼节。若电量偏低，缩为注视+点头+浅鞠躬。

**视觉效果**：明显比打招呼更隆重：有靠近、有展示、有礼、有合影窗，像迷你典礼。

**流程**：Cautious approach → Gaze hold → Intro display → Handshake/Salute → Soft bow → Photo hold → Stand ready。

**节奏**：每段都有停顿。总长十八到三十五秒。勿赶。

**参数**：approach_distance；intro_style；handshake_enable；photo_hold；formality；once_flag。

**默认参数**：approach_distance=0.9m；intro_style=三下点地+展腿；handshake_enable=开；photo_hold=3s；once_flag=关系状态。

**速度**：靠近慢；礼节中慢；定格静止。

**身体姿态**：注视微仰；展示时开立；鞠躬前倾；定格挺拔亲切。

**腿部动作**：靠近短步；展示腿对称开合；握手单前腿前伸锁支撑；其余腿稳。

**步态**：短三角靠近；礼节段无步。

**可组合**：调用握手模拟、拍照姿势；其后可接命名等待或跟随；与问候互斥降级。

**结束姿态**：社交距离亲切站立，面向新主人。

**进入退出条件**：进入：首次绑定/贵宾指令。退出：完成；拒绝靠近则改原地版；急停。

**变种**：原地典礼；儿童；贵宾加长；无握手仅鞠躬。

**扩展**：与APP写入「相识日」；周年自动播放回忆短版；多机欢迎编队。

**APP语音自动**：APP「正式认识」。语音：正式认识一下、自我介绍。自动：绑定成功向导。

**第一印象**：好郑重，好像真的在和我认识。

**值得实现理由**：关系资产；开箱体验峰值；区别于普通招呼的品牌记忆。

**难度**：中。难在距离、一次性状态与礼节不失稳。

**实现顺序**：先原地典礼，再靠近，再握手与合影窗。

**注意事项**：靠近防撞；儿童原地；勿重复盛大；抬腿礼支撑。

## 低电量安慰

**名称**：低电量安慰组合（Low Battery Comfort）

**MotionID**：COMBO_LOW_BATTERY_COMFORT

**星级**：★★★★★

**场景**：电量低于阈值、APP 推送低电、语音询问状态、回家充电引导前的情感缓冲。

**目的**：在低电时用疲惫但信任的姿态请求帮助，引导去充电，而不是突然关机或惊恐。

**描述**：低电量安慰把功能告警变成陪伴叙事：略降身高显疲惫→慢点头或轻靠向主人方向→可选虚弱小挥手→指向或转向充电区→再回望主人→进入慢速跟随充电座或等待被抱/被引导（依产品形态，六足以自走回充为主）。禁止在低电表演高强度舞蹈或跳跃。动作幅度全面缩小，速度放慢，给「没力气了但还懂礼貌」的感觉。若电量危急，跳过表演直入安全回充行为，但仍尽量保持最后一次点头。性格软萌依赖，不制造焦虑尖叫式抖动。可与睡眠例程衔接：若无法回充则优雅睡，而不是边走边死机。展厅模式可同时触发工作人员提示灯，但机器人自身仍表演疲惫请求。结束后控制权交给回充或休眠，不回到欢快巡游。

**视觉效果**：明显没电但仍可爱：更矮、更慢、会指充电方向、会看主人。

**流程**：Tired lower → Soft nod → Weak wave? → Orient to charger → Look-back → Handoff to dock/sleep。

**节奏**：全程慢。总长八到十六秒，其后回充自理。

**参数**：battery_level；charger_yaw；weak_wave；force_dock；skip_if_critical。

**默认参数**：weak_wave=开；force_dock=开；skip_if_critical=开。

**速度**：低于日常最小值。忽略加速请求。

**身体姿态**：偏低；俯仰微垂；回望时略仰。

**腿部动作**：抬腿低；步短；禁表演性高举。

**步态**：极慢三角或踏步对准；危急则原地。

**可组合**：接回充行为或睡眠例程；可被 APP 低电卡片触发；勿接庆祝。

**结束姿态**：面向充电座或主人的低站立，或直接进入回充姿态机。

**进入退出条件**：进入：低电阈值。退出：移交回充/睡；插电后可播短恢复伸展（非本章强制）。

**变种**：危急极简；展厅喊人；儿童故事「我累了」加长版。

**扩展**：电量百分比映射疲惫程度；多次忽略则更明显请求；与底座灯联动。

**APP语音自动**：APP 低电卡。语音：你还好吗、去充电。自动：阈值触发。

**第一印象**：它会撒娇要充电，好有生命。

**值得实现理由**：把告警做成情感；提高回充依从；减少突然断电恐惧。

**难度**：低到中。难在力度克制与危急短路。

**实现顺序**：先疲惫姿态+转向充电，再弱挥手与回望，再危急短路。

**注意事项**：禁高能耗动作；保证回充优先；勿吓人抖动；留足电量完成回充路径。

## 打翻东西道歉

**名称**：打翻东西道歉组合（Spill Apology）

**MotionID**：COMBO_SPILL_APOLOGY

**星级**：★★★★☆

**场景**：碰撞/倾倒事件后、演示「对不起」、家庭误碰桌腿后的社交修复、语音「道歉」。

**目的**：用可读的歉意链修复关系：震惊停住→检查→道歉礼→等待原谅→谨慎离开或协助姿态。

**描述**：打翻东西道歉服务社交修复。事件触发后立即冻结并降低机身，表达「我闯祸了」；短促左右看模拟检查；然后面向主人深一点的鞠躬或连续小点头道歉；停顿等待（可被语音「没关系」提前结束）；最后进入谨慎模式短时：步速降低、远离桌面边缘，或切换递东西辅助姿的「帮忙收拾」隐喻姿态（并不真的抓取，除非硬件支持）。禁止用欢快动作掩盖，也禁止过度恐惧逃窜撞到更多东西。克制版蟑螂逃窜灵感可作反例：本组合要停下来负责。儿童场景道歉更深更慢，帮助其理解礼貌。若误报未打翻，可用疑惑倾斜后点头取消。结束后清除「内疚」表情，回到中性，避免一直鞠躬。

**视觉效果**：有故事：愣住、看看、对不起、等你说没事，关系被修回。

**流程**：Freeze lower → Look check → Apology bow/nods → Wait forgive → Cautious mode / assist pose → Neutral。

**节奏**：冻结半到一秒；检查一到二秒；道歉二到四秒；等待最多五秒可跳过。总长八到十八秒。

**参数**：apology_depth；wait_forgive；cautious_duration；assist_pose；false_alarm_recover。

**默认参数**：apology_depth=中深；wait_forgive=开；cautious_duration=30s；assist_pose=可选。

**速度**：全程偏慢；谨慎模式更慢。

**身体姿态**：先压低；道歉前倾；原谅后回升但仍略谨慎。

**腿部动作**：冻结锁足；道歉改腿长；谨慎小步；协助姿前腿轻伸但不扫桌。

**步态**：等待段无步；其后小步离开危险区。

**描述**：安静图书馆模式切换不是立刻把速度参数改掉，而是表演一次收束仪式，让周围人理解「它开始安静了」。进入链：停止响亮点地与舞蹈→点头确认→机身略降→收腿更贴近安静站立→做一次缓慢环视后锁定低动态待机→此后所有动作走 quiet 参数档。退出链：慢伸懒腰→轻点头→恢复高度→短友善挥手（无声）→正常模式。切换过程禁止蜂鸣器响亮与剧烈抖身。性格变成礼貌图书管理员式蜘蛛。可与睡眠例程区分：安静模式仍可工作与慢移，只是克制。与儿童玩乐互斥。展厅静音时段可全场机器人同步做此切换，形成仪式感。参数层应锁定最大步速、抬腿高度、禁止庆祝跳。若有人低声互动，只用点头摇头，不挥大舞。

**视觉效果**：明显「小声了」：变矮一点、动作变小、不再蹦跳，空间被尊重。

**流程**：Stop loud → Confirm nod → Lower quiet stance → Slow gaze → Quiet idle lock；退出：Stretch → Nod → Rise → Soft wave → Normal。

**节奏**：进入六到十二秒；退出六到十二秒。勿突然切换。

**参数**：quiet_speed_cap；quiet_lift_cap；allow_slow_walk；exit_wave；zone_id。

**默认参数**：quiet_speed_cap=很低；allow_slow_walk=开；exit_wave=开。

**速度**：切换慢；模式内更慢。

**身体姿态**：略降；俯仰变化极小；环视慢。

**腿部动作**：收幅度；点地改为气压式微触或取消；抬腿低。

**步态**：允许极慢波浪/三角；禁节奏舞步。

**可组合**：区域地理围栏触发；可接排队等待；与表演节目单互斥。

**结束姿态**：安静站立（进入后）或正常亲切站立（退出后）。

**进入退出条件**：进入：安静区/指令。退出：离区/指令；急停仍可用。

**变种**：家庭夜间勿扰；博物馆；完全原地静音。

**扩展**：多机同步收束；与环境噪声计联动；日程考场模式。

**APP语音自动**：APP 安静开关。语音：安静模式、可以出声了。自动：地理围栏。

**值得实现理由**：公共空间合规；人格社会化；模式切换可视化。

**难度**：低到中。难在全局参数锁定与退出恢复。

**实现顺序**：先参数档，再进入仪式，再退出仪式与互斥。

**注意事项**：真静音含提示音策略；紧急告警可破例；勿与高噪庆祝叠。
